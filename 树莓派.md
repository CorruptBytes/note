# 树莓派引脚

> 引脚是树莓派与外界交互的物理接口，树莓派上具有多种引脚。

- 查看引脚信息

  ```bash
  pinout
  ```

  > 注意:
  >
  > - 这种方法可以查看树莓派相关信息以及引脚信息。

## 分类

- 电源引脚

  > 可提供三种电压:5v,3.3v,GND(Ground)

- 常规GPIO控制引脚

  > 可以通过编写程序控制这些引脚的高低电平

- 特殊GPIO通讯引脚

  > 可用作SPI通讯，I2C通讯，TXD/RXD串口通讯

## 引脚编码

> 为了区分不同引脚，对引脚进行编码。树莓派引脚有三种编码方式:

- 板载编码

  > 按照树莓派主板上引脚排列的顺序对引脚进行编号

  ![image-20250622150814623](D:\笔记\图片\image-20250622150814623.png)

  > 注意:
  >
  > - 通过pinout指令查出的引脚信息便是以这种方式编码。

- BCM编码

  > 参考BCMSoc的通道编号侧重CPU寄存器，使用Python语言控制引脚时，常使用BCM编码

- WiringPi编码

  > 把GPIO端口从0开始编码。使用C语言控制树莓派引脚时，使用WiringPi编码

## WiringPi库

> 操作树莓派GPIO引脚进行编程的c语言库，同时也提供了一系列命令用于查询引脚相关信息的命令。现在官方已经停止维护，由第三方开发者自行更新维护。

### 安装

1. 下载WiringPi库的源码

   ```
   git clone https://github.com/WiringPi/WiringPi.git
   ```

2. 运行源码中的build文件进行编译

   ```bash
   cd WiringPi
   ./build
   ```

   

### 命令

> WiringPi提供的命令均已gpio开头

- gpio -v

  > 查看当前WiringPi库的版本以及相关信息

- gpio -readall

  > 查看引脚的相关信息，包括板载编码，BCM编码，WiringPi编码等信息

- gpio [-g] mode [port] [mode]

  > 设置引脚模式，-g选项代表使用BCM编码

  ```
  gpio -g mode 4 out
  #意为将BCM编码的4号引脚设置为输出模式
  ```

- gpio [-g] read [port]

  > 读取端口的状态

- gpio [-g] write [port] [status]

  > 设置引脚状态，1表示高电平，0表示低电平

## GPIO Zero库

> 官方提供的控制GPIO引脚的Python库，用于控制GPIO引脚的状态与功能。简单易用，适合初学者

# 串行通讯接口

> 简称串口(UART),特点是数据一位一位的按照顺序传送。树莓派的外设包含两个串口:硬件串口与mini串口。
>
> 在树莓派中，引脚8与引脚10是作为中串口的收发端，引脚8为发送端，被称为TXD(Transmit Data);引脚10为接受端，被称为RXD(Receive Data)。

- 优点

  > 通信线路简单，成本低，适合远距离通信

- 缺点

  > 传输速度较慢

## 分类

- 主串口

  > 可以通过引脚实现通讯

- 辅助串口

  > 不能使用GPIO引脚通信，默认分配给蓝牙端

> 注意:
>
> - 默认情况下，mini串口是禁用的，无论将它指定为主UART还是辅助UART。

## 硬件串口

> 由硬件实现，有单独的波特率，时钟源，性能高，传输可靠

## mini串口

> 性能较低，功能较简单

# 小知识

## SOC

> System On Chip，系统级芯片，也称为"片上系统"。是一种将**计算系统中的多个核心功能模块集成在一颗芯片上**的技术方案。它是现代电子产品（尤其是智能手机、平板、嵌入式设备，单板计算机等）的**核心组件**。

- 优点

  |        作用        |                             说明                             |
  | :----------------: | :----------------------------------------------------------: |
  |   🎯 **功能集成**   | 把 CPU、GPU、内存控制器、I/O 接口、通信模块等整合到一颗芯片上，代替多个独立芯片 |
  |   ⚡ **降低功耗**   |      模块内部通信比芯片间通信能耗更低，更适用于便携设备      |
  |   📏 **减小体积**   |     精简电路板设计，使设备更小更轻，例如智能手表、树莓派     |
  |   💰 **降低成本**   |           减少元件数量和布线复杂度，提升产能和良率           |
  |   🚀 **提高性能**   |           多模块协同优化，数据交换更快，响应更及时           |
  | 📡 **支持专用功能** | 可内置 AI 加速器、图像处理器、无线通信模块等，满足特定应用需求 |

- 可包含模块

  |     模块     |              作用              |
  | :----------: | :----------------------------: |
  |     CPU      |  执行通用计算任务（主控单元）  |
  |     GPU      |  图形渲染、图像处理、并行计算  |
  |  DSP / NPU   | 数字信号处理（如音频）、AI加速 |
  |  RAM 控制器  |    管理与外部内存的数据传输    |
  |   存储接口   |    支持 eMMC、UFS、SD 卡等     |
  |   通信模块   |  Wi-Fi、蓝牙、LTE、5G、GPS 等  |
  |  外设控制器  |    UART、SPI、I2C、GPIO 等     |
  | 电源管理模块 |   控制芯片内外供电，优化能耗   |
  |   安全模块   | 支持加密、安全启动、身份验证等 |
