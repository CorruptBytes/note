## 删除链表元素

> 在高级语言中，有垃圾回收机制，只要将链表的引用改变，会自动回收链表节点内存。但是要注意:删除头节点和非头节点的处理是不同的。

- 原链表删除元素

```java
public ListNode deleteNode(ListNode head, int val) {
		//删除头节点
        while (head != null && head.val == val) {
            head = head.next;
        }
    	//删除非头节点元素，此时头节点已经处理完成，只需要遍历后续链表即可
        ListNode currentNode = head;
        while (currentNode != null && currentNode.next != null) {
            if (currentNode.next.val == val) {
                currentNode.next = currentNode.next.next;
            } else {
                currentNode = currentNode.next;
            }
        }
    //最后返回链表的头节点
    return head;
}
```

- 具有虚拟头节点的链表删除元素

  > 为了统一对链表各种节点的处理，可以为链表增加一个虚拟的头节点

  ```java
   	//使用虚拟头节点
      public ListNode deleteNode2(ListNode head, int val) {
          
          ListNode virtualHead = new ListNode(0,head);
  
          ListNode currentNode = virtualHead;
          while (currentNode.next != null) {
              if (currentNode.next.val == val) {
                  currentNode.next = currentNode.next.next;
              }else {
                  currentNode = currentNode.next;
              }
          }

          return virtualHead.next;
      }
  ```

  

# 队列

## 单调队列（Monotonic Queue）

> 单调队列是一种队列结构，队列中的元素始终保持**单调性**，即单调递增或单调递减。

### **特性**

- 队列中的元素满足单调性，始终保持：
  - **单调递增队列**：队首最小，队尾最大
  - **单调递减队列**：队首最大，队尾最小
- 队列的**队首元素永远是当前区间（窗口）的最值**。

### **原理**

单调队列的核心操作是：

> **当往队尾插入新元素时，如果新元素破坏了队列的单调性，则将队尾所有破坏单调性的元素弹出，再插入新元素。**

单调队列本质是一个最值候选列表，以**单调递减队列**为例（队首最大）：

- 队首永远是当前队列最大值
- 队列中的其他元素是未来可能成为最大值的候选
- 插入新元素时，如果前面的元素不可能再成为队列中的最大值，则直接清理掉它们

### **应用场景**

- **滑动窗口最值问题**：在窗口大小为 k 的滑动窗口中，单调队列可以快速获取每个窗口的最小值或最大值
- **其他最值优化问题**：如前缀和最短区间问题、动态规划优化、Next Greater Element 等

## 优先级队列

> 优先级队列中的元素都有一个优先值，出队时，优先级高的元素先出

- 特性

  | 特性                         | 说明                                                   |
  | ---------------------------- | ------------------------------------------------------ |
  | ✅ 元素带有“优先级”           | 每个元素都有一个优先级值，决定它的出队顺序             |
  | ✅ 弹出的是“优先级最高”的元素 | 与普通队列（FIFO）不同，优先级高的元素先出             |
  | ✅ 支持动态插入               | 插入新元素后仍保持优先级顺序                           |
  | ❌ 不保证顺序遍历             | 遍历整个队列时不能保证元素的顺序一致性（除非手动排序） |

### 实现

> 优先级队列通常使用堆来实现

- Java

  > Java中的PriorityQueue就是一个优先级队列，默认使用小顶堆实现(小的先出),可以通过 Comparator 改成大顶堆。

# 二叉树

## 分类

- 满二叉树

  > 所有叶子节点的度要么为0，要么为2。如果满二叉树的深度为k，则节点数量为`2**k - 1`

- 完全二叉树

  > 除最后一层后与满二叉树一样，最后一层的节点是从左到右连续的。满二叉树一定是完全二叉树

- 二叉搜索树(Binary Search Tree)

  > 左子树的所有节点一定小于中间节点，右子树的所有节点一定大于中间节点。常用于元素搜索，时间复杂度为`logn`

  - 中序遍历二叉搜索树，得到的是一个递增的序列。

- 平衡二叉搜索树

  > 满足二叉搜索树的条件，且左子树与右子树的高度差的绝对值不超过1

## 实现

> 根据存储方式的不同，可以分为链式存储和顺序存储

- 链式存储

  > 类似链表，左指针与右指针来连接二叉树的多个节点

  ```java
  public class TreeNode {
      public int val;
      public TreeNode left;
      public TreeNode right;
      public TreeNode(int val){
          this.val = val;
      }
  }
  ```

- 顺序存储

  > 使用数组存储二叉树，把二叉树的节点按逐层从左到右存入数组，节点的左孩子为`2 * i +1`，右孩子为`2 * i + 2`。顺序存储不常使用。

## 遍历

> 前序，中序，后序均属于深度优先搜索(DFS),层序遍历属于广度优先搜索()
>
> 二叉树的前中后序遍历既可以使用递归遍历，也可以通过栈进行迭代来模拟递归遍历

- 前序遍历

  > **先访问根节点，再递归遍历左子树，最后递归遍历右子树**。即根左右

  ```java
  //迭代法遍历
  public List<Integer> preorderTraversal(TreeNode root) {
     	Stack<TreeNode> stack = new Stack<>();
  	List<Integer> result = new ArrayList<>();
  	stack.push(root);
  	TreeNode currentNode;
  	while (!stack.isEmpty()) {
  		currentNode = stack.pop();
  		if (currentNode != null) {
  			result.add(currentNode.val);
  		} else {
  			continue;
  		}
  		stack.push(currentNode.right);
  		stack.push(currentNode.left);
  	}
  	return result;
  }
  ```

  

- 中序遍历

  > **先递归遍历左子树，再访问根节点，最后递归遍历右子树**。即左根右

  ```java
   public List<Integer> inorderTraversal(TreeNode root) {
              List<Integer> result = new ArrayList<>();
              Stack<TreeNode> stack = new Stack<>();
              TreeNode currentNode = root;
              while (!stack.isEmpty() || currentNode != null) {
                  if (currentNode != null) {
                      stack.push(currentNode);
                      currentNode = currentNode.left;
                  } else {
                      currentNode = stack.pop();
                      result.add(currentNode.val);
                      currentNode = currentNode.right;
                  }
              }
              return result;
          }
  ```

  

- 后序遍历

  > **先递归遍历左子树，再递归遍历右子树，最后访问根节点**。即左右根

  ```java
  //递归遍历
  public List<Integer> postorderTraversal(TreeNode root) {
              List<Integer> result = new ArrayList<>();
              postorderTraversal(root,result);
              return result;
          }
  private void postorderTraversal(TreeNode node,List<Integer> list) {
            if (node == null) {
                return;
            }
            postorderTraversal(node.left,list);
            postorderTraversal(node.right,list);
            list.add(node.val);
          }

  		stack.push(currentNode.left);

  public List<List<Integer>> levelOrder(TreeNode root) {
              LinkedList<TreeNode> queue = new LinkedList<>();
              List<List<Integer>> result = new ArrayList<>();
              if (root != null) {
                 queue.addLast(root);
              }
              while (!queue.isEmpty()) {
                  int size = queue.size();
                  List<Integer> list = new ArrayList<>();
                  while (size != 0) {
                      TreeNode node = queue.removeFirst();
                      list.add(node.val);
                      if (node.left != null) {
                          queue.addLast(node.left);
                      }
                      if (node.right != null) {
                          queue.addLast(node.right);
                      }
                      size--;
                  }
                  result.add(list);
              }
              return result;
          }
  ```
  

## 二叉树深度与高度

>
> 二叉树的高度即当前节点到叶子节点的距离

## 根据遍历顺序构造二叉树

> 知道一颗二叉树的前序遍历与中序遍历可以构造出唯一一颗二叉树，知道二叉树的后序遍历和中序遍历也可构造出唯一一颗二叉树。知道前序和后序不可以唯一构造出一颗二叉树

- 中序与后序

  > 递归处理，将每一个节点视为根节点,通过后序数组的最后一个元素可以知道根节点的值，通过在中序数组中寻找根节点可以分隔出根节点的左孩子节点与右孩子节点的中序遍历数组，再通过分隔后数组的长度从后序数组分隔出左孩子和右孩子的后序遍历数组。

  ```java
  	public TreeNode buildTree(int[] inorder, int[] postorder) {
          return traversal(inorder,postorder);
      }
      public TreeNode traversal(int[] inorder, int[] postorder) {
          if (postorder.length == 0) {
              return null;
          }
          int rootValue = postorder[postorder.length - 1];
          TreeNode root = new TreeNode(rootValue);
          if (postorder.length == 1) {
              return root;
          }
          int index = 0;
          for (; index < inorder.length; index++) {
              if (inorder[index] == rootValue) {
                  break;
              }
          }
          root.left = traversal(Arrays.copyOfRange(inorder,0,index),Arrays.copyOfRange(postorder,0,index));
          root.right = traversal(Arrays.copyOfRange(inorder,index + 1,inorder.length),Arrays.copyOfRange(postorder,index,postorder.length - 1));
          return root;
      }
  ```

              return null;

              return treeNode.left;
          } else if (treeNode.right != null && treeNode.left == null) {

          } else {

# 树

## `B-Tree`

> B树是一种多叉平衡搜索树，相比于二叉平衡搜索树，它在数据量不变的情况下，压缩了树的高度，从而降低了查找次数

- B树一个节点中可以存储多个元素，且一个节点可以有多个子节点。
- B树一个节点最多可以有`n`个子节点，称为`N阶B树`。

**特点**

- **平衡:**B树的所有叶子节点均位于同一层
- **有序：**节点内元素有序，且节点内元素的左子树均小于此元素，右子树均大于此元素。
- **多路：**对于n阶B树，其节点最多有n个子节点，节点内最多有`n-1`个元素。且要求根节点最少有2个子节点，1个元素；其他节点最少有`⌈2/n⌉`个子节点，`⌈2/n⌉ - 1`个元素。

## `B+Tree`

> `B+Tree`性质基本与`B-Tree`一致，不过`B+Tree`的非叶子节点仅存储用于查找记录的关键字(`key`)，不存储具体数据。

- 与`B-Tree`一致，一个节点可存储多个`key`。

- `B+Tree`的叶子节点与非叶子节点结构不同，非叶子节点仅存储用于查找的关键字(key)，而叶子节点不仅存储关键字(key)，还存储指向此关键字对应记录的指针。
- `B+Tree`的所有非叶子节点关键字最终都会出现在叶子节点中。
- 因此`B+Tree`查找数据时，一定会走到叶子节点才会结束。所有叶子节点连接成一个有序单向链表(或双向链表)用于进行范围搜索。

**如果 B+Tree 不是用来存记录，只用来排序数字，则叶子节点不用存指向数据的指针，此时叶子节点与非叶子节点结构一致**

## 前缀树(`Trie`)

> 用于高效地存储和检索字符串数据集的一种树形数据结构。

**用途**

- 智能补全
- 拼写检查

### 实现

父节点保存子节点数组，直接将数组下标映射为字符，每一层存储字符串的一个字符。通过`boolean`字段判断此节点是一个字符串的结尾。

```java
class Trie {
    private Trie[] children;
    private boolean isEnd;
	public Trie() {
        children = new Trie[26];
        isEnd = false;
    }
    
    public void insert(String word) {
        Trie node = this;
        for (int i = 0; i < word.length(); i++) {
            char ch = word.charAt(i);
            int index = ch - 'a';
            if (node.children[index] == null) {
                node.children[index] = new Trie();
            }
            node = node.children[index];
        }
        node.isEnd = true;
    }
    
    public boolean search(String word) {
        Trie node = searchPrefix(word);
        return node != null && node.isEnd;
    }
    
    public boolean startsWith(String prefix) {
        return searchPrefix(prefix) != null;
    }

    private Trie searchPrefix(String prefix) {
        Trie node = this;
        for (int i = 0; i < prefix.length(); i++) {
            char ch = prefix.charAt(i);
            int index = ch - 'a';
            if (node.children[index] == null) {
                return null;
            }
            node = node.children[index];
        }
        return node;
    }
}

```

## 红黑树

红黑树(`Red-Black Tree`)是一种自平衡的二叉搜索树，**可在动态插入、删除时，始终保证树的高度是 log n**，从而让查找、插入、删除的效率都稳定在 **O(log n)。**

<h4>特性</h4>

红黑树首先是一种二叉搜索树，因此满足左子树的节点均小于根节点，右子树的节点均大于根节点。在此基础上，红黑树通过颜色标记节点,且每一棵红黑树都必须满足：

1. **每个节点非红即黑**
2. **根节点是黑色**，**所有叶子节点(特指`null`）是黑色**
3. **红色节点不能相邻**
4. **黑高一致性：**从任一节点到其所有后代叶子节点的路径上，黑色节点数量相同

最后两条性质保证了红黑树中的**最长路径不超过最短路径的 2 倍**

相比于`AVL`树任一节点左右子树的高度差不超过1的严格平衡，红黑树仅保证树的近似平衡。因此红黑树的查找效率略逊于`AVL`树，但插入删除时的平衡操作少，因此插入和删除效率更高。

<h3>插入规则</h3>

红黑树在插入新节点时，**首先按照二叉搜索树的规则完成插入**。且新插入的节点**默认为红色**，因为：

- 若新节点为红色，**只可能违反**：
  - 根节点必须为黑色
  - 不允许出现红色节点相邻（红红冲突）
- **不会破坏黑高一致性**

相比插入黑色节点，**对整棵树结构的影响更小**，调整代价也更低。

当插入新节点导致**颜色约束被破坏**时， 红黑树会通过**颜色翻转和旋转两种操作**来调整。

插入节点为根节点，则只需将插入节点翻转为黑色即可。

插入节点非根节点，因为新节点默认为红色，则仅当出现“红红冲突”（父节点为红）时才需要调整红黑树，此时需要根据叔叔节点颜色决定调整策略。

<h4>叔叔节点为红</h4>

无需改变树结构，父节点和叔叔节点染黑，祖父节点染红。进而将祖父节点作为插入节点，继续修复。

- 本质是将红红冲突被**向上转移**，可能需要**递归向上继续修复**。

<h4>叔叔节点为黑</h4>

<h3>删除规则</h3>

红黑树在删除节点时，**首先按照二叉搜索树的规则完成删除**。

- 目标节点为叶子节点，直接删除
- 目标节点只有一个子节点（左子树或右子树）：删除目标节点，用其**唯一子节点**顶替原来的位置
- 目标节点同时存在左右子树：找到其中序遍历的前驱节点(左子树中最大的节点)或后继节点(右子树中最小的节点)，用该节点的 **值 / 数据替换目标节点**，再删除该前驱 / 后继节点(该节点至多只有一个子节点,如果存在子节点则删除后由子节点顶替它)。
  - 实际删除操作最终**又落在「叶子节点」或「只有一个子节点的节点」上**

二叉搜索树的实际删除操作最终**一定会落在「叶子节点」或「只有一个子节点的节点」上**，所以只需要考虑两种情况：

<h4>目标节点只有一个子节点</h4>

由于红黑树自身性质的约束，此时只可能存在一种情况：被删除节点 **必为黑色**，其唯一子节点 **必为红色**，且该红色子节点 **一定是叶子节点**。

## 堆

堆是一种有序的完全二叉树

### 分类

**大根堆**

堆中的任一节点大于等于其左右孩子节点，所以堆中的根节点一定是最大元素。

**小根堆**

堆中的任一节点一定小于等于其左右孩子，所以堆中发根节点一定是最小元素。

### 存储方式

**顺序存储**

由于堆是一个完全二叉树，他采用顺序存储时所有的元素都是连续的，不会存在空间浪费，因此较为适合顺序存储。

将堆中的节点编号，根节点编号为1，每个节点其左孩子编号为`2 * p`，右孩子为`2 * n + 1`。编号即为节点的存储下标。除了根节点，其他节点的父节点的编号为` c / 2`

- 通常从数组的下标1开始存储节点，跳过下标0

**链式存储**

### 堆排序

 - 堆排序不稳定。

#### 建堆

从最后一个非堆节点开始调整，将每一个节点及其子节点都调整为堆。由于叶子元素本身即为堆，因此从最后一个非叶子节点开始调整。

大根堆的堆顶元素是最大值，每次将堆顶元素与堆中最后一个元素交换，然后忽略此元素，调整堆顶元素，使其符合堆的定义，重复 `n - 1`次。


# 图

图是一种表示节点之间关系的非线性数据结构，由顶点集合和边集合组成。

<h4>度(<code>Degree</code>)</h4>

度是顶点的属性。

在无向图中，顶点的度是指与该顶点相关联的边的条数。

在有向图中，度分为两种：

- **入度：**指向该顶点的边的数量
- **出度：**从该顶点发出的边的数量

<h4>连通性</h4>

在无向图中，如果图中任意两个顶点之间都存在路径，则称该图为**连通图**，否则为非连通图

在有向图中，图的连通性分为两种：

- 若任意两个顶点之间都存在双向路径，则称为强连通图
- 若忽略方向后变成无向图是连通的，则称为弱连通图。

**连通分量**

无向图中的极大连通子图称为连通分量，一个连通图中有一个连通分量，也就是它本身。

## 图的存储方式

<h3>邻接矩阵</h3>

用一个二维数组（n×n，n 是顶点数）表示顶点之间的连接关系。`matrix[i][j]`的值表示顶点 i 和顶点 j 之间的连接关系，如1表示有边，0表示无边，也可以存储边的权值。

- 对于无向图，`matrix[i][j]`和``matrix[j][i]``才表示一条完整的边，所以无向图的邻接矩阵是对角线对称的。

**优点**

- **查询快**：判断两个顶点是否相连只需要 O(1)的时间复杂度
- **实现简单**：二维数组直接表示

**缺点**

- **占用空间大**：O(n²)，不适合顶点多而边少的稀疏图

<h3>邻接表</h3>

用一个 **一维数组** 存储图的所有顶点，每个顶点对应一个**链表或数组**，存储它的**邻居顶点**和与邻居的连接信息。

**优点**

- **节省空间**：O(V + E)，适合稀疏图（边少顶点多）
- 遍历顶点邻居快

**缺点**

- 判断两点是否相连稍慢（需要遍历链表）

## 图的遍历

图的遍历方式可分为两种：深度优先搜索(DFS，Depth-First Search)和广度优先搜索(BFS，Breadth-First Search)

<h3><code>DFS</code></h3>

从某个顶点出发，沿着一条路径尽可能深入，直到无法继续再回溯，遍历所有顶点。

**特点**

- 沿着一条路径尽可能深入，直到到达终点或遇到已访问节点后回溯 
- 依赖栈结构实现（递归或手动栈）
- 适合**查找路径、拓扑排序、连通分量**

```java
DFS(vertex v):
    标记 v 为已访问
    for 每个邻居 u of v:
        if u 未被访问:
            DFS(u)
                
                
DFS(start):
    创建栈 stack
    stack.push(start)
    标记 start 为已访问

    while stack 不为空:
        v = stack.pop()
        for 每个邻居 u of v:
            if u 未被访问:
                标记 u 为已访问
                stack.push(u)
```



<h3><code>BFS</code></h3>

从某个顶点出发，先访问它的所有邻居，再访问邻居的邻居，逐层向外扩展，直到所有顶点访问完成。

**特点**

- 队列结构实现
- 适合**最短路径（无权图）、层次遍历**

```java
BFS(vertex start):
    初始化队列 Q
    将 start 入队并标记为已访问
    while Q 非空:
        v = Q.dequeue()
        for 每个邻居 u of v:
            if u 未被访问:
                标记 u 已访问
                Q.enqueue(u)
```

# 并查集

并查集是一种树型的数据结构，用于处理一些不交集（Disjoint Sets）的合并及查询问题。

<h3>原理</h3>

并查集将每一个集合抽象为一个树形结构，将集合中的一个元素作为根结点，其他所有集合内元素均是此元素的子结点，其中子结点指向其根结点，根节点指向自己。多个集合就形成了森林的数据结构。这样，就可以通过根节点元素唯一确认一个集合。

在实际存储中，通常对集合元素进行连续编号，通过一个数组保存所有集合，其中数组的下标对应元素的编号，此位置的值对应此元素的父结点的编号。

因此，根节点的索引值与存储值相同，非根节点的索引值与存储值不同。这样就可以快速区分根节点和非根节点。

<h3>基础操作</h3>

在并查集数据结构的基础上，要实现两个基础操作：`Find`(查询)操作，`Union`(合并)操作。

<h4><code>Find</code></h4>

对于查询操作，只要寻找元素的根节点，也就是数组编号位置所存储的编号值就可以确定元素位于哪个集合。

<h4><code>Union</code></h4>

对于合并操作，只需要将其中一个集合的根节点设置为另一个集合的子结点即可，这样此根节点之下的所有元素均挂载到了另一个根节点下，也就是两个集合合并在一起。

在实际操作中，就是将被合并的其中一个集合的根节点所在位置设置为另一个集合的根节点编号值。

<h3>实现</h3>

```java
public class Dsu {
    private int[] set;

    public Dsu(int capacity) {
        set = new int[capacity];
        initialize();
    }

    public void initialize() {
        for (int i = 0; i < set.length; i++) {
            set[i] = i;
        }
    }
    public int find(int e) {
        if (set[e] == e) {
            return e;
        } else {
            return find(set[e]);
        }
    }

    public void union(int e1, int e2) {
        int r1 = find(e1);
        int r2 = find(e2);
        if (r1 == r2) return;
        set[r1] = r2;
    }
}
```

**路径压缩**

```java
 //优化find，由于合并后，会出现多层树形结构，因此在查找的回溯过程中将非直接子结点均直接指向根节点。从而压缩查找路径，这称为路径压缩。
    public int find1(int e) {
        if (set[e] == e) {
            return e;
        } else {
            return set[e] = find1(set[e]);
        }
    }
```

<h4>合并操作优化</h4>

**按秩合并**

按秩合并分为两种：按高度合并和按大小合并

当高度较高的树合并到高度较低的树，会使树形结构增高，而高度较低的数组合并到高度较高的数组上，不会使高度发生变化。因此可以维护一个数组，记录各个根节点的高度，从而对合并操作进行优化，也就是尽量减少树的高度，使查找效率更高

```java
    public void union(int e1, int e2) {
        int r1 = find(e1);
        int r2 = find(e2);
        if (r1 == r2) return;
        if (height[r1] > height[r2]) {
            set[r2] = r1;
        } else if (height[r1] < height[r2]){
            set[r1] = r2;
        } else {
            set[r1] = r2;
            height[r2]++;
        }
    }
```

- 路径压缩可能会改变树的高度，此时高度数组记录的仅是树的估计高度，而不是实际高度。
- 在不考虑路径压缩的情况下，这种合并方式可以将树的高度控制在(log n)以内。

**按集合大小合并**

另一种合并思路是通过数据记录每一个集合的元素个数，将较小的集合合并到较大的集合上。

- 在不考虑路径压缩的情况下，这种合并方式可以将树的高度控制在(logn)以内。

# 布隆过滤器

布隆过滤器（`Bloom Filter`）是一种用来判断“某个元素是否可能存在”的概率型数据结构。

<h4>特点</h4>

- **查询速度极快**（O(k)，k=哈希函数个数）
- **极省内存**，非常适合大规模数据
- 安全性高，通过多个`bit`位标识数据，不直接存储原始数据。
- 存在误判的情况：布隆过滤器判断存在的元素未必存在，但判断不存在的元素一定不存在。
- 无法删除元素：多个元素可能共享同一个bit。

<h3>原理</h3>

布隆过滤器由一个位数组和多个哈希函数组成。通过多个哈希函数将元素映射到位数组中的多个位置，查询时只要有一个位置为0就一定不存在，否则可能存在。

**添加元素**

插入元素时，对元素分别使用多个哈希函数计算出多个位图下标，将这些下标置为1。

**查询元素**

查询元素时，只要使用同样的多个哈希函数计算出多个位图下标，如果这些位置均为1，则元素可能存在；如果有一个是0，则元素一定不存在。

- 扫描到数字，直接放入后缀表达式中
- 扫描到`( [ {`，直接放入栈中；扫描到`) ] }`，从栈中依次弹出元素放入后缀表达式，直到找到对应的左括号，把左括号弹出，直接消除左右括号
- 扫描到运算符，则依次和栈顶元素比较，如果栈顶元素优先级大于等于当前运算符的优先级，则将栈顶元素弹出放入后缀表达式；反之，直接进栈。(括号的优先级是最高的，当比较到括号时，直接停止)

## 抽象数据结构与具体数据结构

| 概念层级                 | 名称          | 示例                       | 描述                                   |
| ------------------------ | ------------- | -------------------------- | -------------------------------------- |
| 抽象数据结构（ADT）      | 抽象接口/行为 | 优先级队列、栈、队列、集合 | 定义了“支持什么操作”但不关心“怎么实现” |
| 具体数据结构（实现方式） | 数据结构实现  | 数组、链表、堆、树、哈希表 | ADT 的具体实现方式                     |

> 抽象数据结构只定义特性和行为，它是由具体的数据机构实现
