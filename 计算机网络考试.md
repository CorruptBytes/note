# 第一章

**计算机网络**

计算机网络（Computer networking）是一个将众多分散的、自治的计算机系统，通过通信设备与通信线路连接起来，由功能完善的网络软件及通信协议实现资源共享和信息传递的系统。简单来说,计算机网络以传输信息为基本目的,用通信线路与通信设备将多个计算机连接起来的计算机系统的集合。

## 电路交换

**代表：传统电话网**

- 通信前：先**建立专用通路**
- 通信中：**独占链路资源**
- 通信后：**释放电路**

✅ 优点：时延稳定
 ❌ 缺点：线路利用率低

## 分组交换

将一个报文划分为等长的数据段，在每个数据段前加入控制信息组成分组。计算机将分组通过通信链路发送给分组交换机。分组交换机采用存储转发的方式：接收到分组后，先将分组暂存，检查首部中的地址查找转发表，找到合适的接口将分组转交给下一个分组交换机。一步步经过多个分组交换机把分组转发到最终的目的计算机。

**分组丢失**

对于每一个与分组交换机相连的链路，该分组交换机内都有一个输出缓存，用于存储分组交换机准备发往那条链路上的分组。当一个分组到达输出缓存时输出缓存已满，则会发生分组丢失，即将到达的分组或已经排队的分组丢弃。

✅ 优点：链路利用率高
 ❌ 缺点：时延不确定、可能乱序

## 五层协议的原理体系结构

**应用层**

负责为用户提供网络应用服务

**传输层**

负责为应用层提供端到端的数据传输服务

**网络层**

网络层负责将网络中的"分组"从源主机转发到目的主机，其核心功能是分组转发和路由选择。

**数据链路层**

数据链路层负责将分组在相邻结点间传递

**物理层**

物理层负责在传输媒体上传输比特流。

例：**WWW 访问一个网页**

## 计算机网络性能指标

**速率**

数据的传输速率，也称为数据率或比特率。单位为bit/s。

**带宽**

表示通道传送数据的能力，即网络中一点到另一点所能通过的最高数据率。单位为bit/s

**吞吐量**

表示在单位时间内通过某个网络的数据量。

**时延**

指数据从网络的一端传送到另一端所需的时间。

总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延

**发送时延** = 数据长度 / 发送速率

**传播时延** = 距离 / 传播速度

**丢包率**

分组在传输过程中丢失的分组数量和总分组数量的比率。

# 第二章：物理层

物理层负责在传输媒体上传送比特流。这一层，定义了传输媒体的类型，位的表示，数据率，位同步，链路配置，物理拓扑结构，传输方式和传输设备。

## 四大特性

1️⃣ **机械特性**

- 接口形状
- 引脚数量
- 引脚排列

2️⃣ **电气特性**

- 电压范围
- 信号电平

3️⃣ **功能特性**

- 各引脚的功能
- 信号的作用

4️⃣ **规程特性**

- 传输顺序
- 时序关系

## 传输媒体

可分为导引型传输媒体和非导引型传输媒体

### 导引型传输媒体

**双绞线**

将两根铜导线按照一定规则绞合构成，以太网常用介质。

- 最常用
- 抗干扰能力一般
- 传输距离短
- 成本低

**同轴电缆**

由内导体铜质芯线、绝缘层、外导体屏蔽层、绝缘保护套层构成。

- 抗干扰能力 **强于双绞线**
- 早期以太网使用
- 现在基本被淘汰

**光纤**

- 传输距离远
- 抗电磁干扰强
- 带宽大
- 成本高

### 非导引型传输媒体

即无线传输，利用电磁波为载体传输数据。

- 不需要物理线路
- 易受干扰

## 网络互联设备

### 集线器（Hub）

#### 工作原理

- **物理层设备**
- 收到信号 → **广播到所有端口**
- 不识别 MAC
- 半双工

#### 关键结论（必考）

- ❌ **不隔绝冲突域**
- ❌ **不隔绝广播域**
- 整个 Hub = **一个冲突域 + 一个广播域**

### 交换机（Switch）

#### 本质

> **交换机 = 多端口网桥**

#### 工作原理

- **数据链路层**
- 根据 **MAC 地址表** 转发
- 支持全双工

#### 关键结论

- ✅ **隔绝冲突域**
- ❌ **不隔绝广播域**

### 路由器（Router）

#### 工作原理

- **网络层设备**
- 根据 **IP 地址** 转发
- 连接不同网络

#### 关键结论

- ✅ **隔绝冲突域**
- ✅ **隔绝广播域**

## 虚拟局域网 VLAN

VLAN 是在交换机上划分的逻辑局域网

- 基于端口 / MAC / 协议
- **逻辑划分，与物理位置无关**

- **交换机本身不隔绝广播域,但 VLAN 可以做到“逻辑隔绝**

### VLAN 的核心作用

- **隔绝广播域**
- 减少广播风暴
- 提高安全性

# 第三章：数据链路层

## MAC 地址

数据链路层使用的、**唯一标识网络接口的硬件地址**。

- 长度：**48 位**
- 由厂商分配
- 写入网卡，**与硬件绑定**

数据链路层负责在相邻结点间传递分组，在同一个局域网中，需要**唯一标识下一跳设备**才能确定传递的链路。

## IP地址

网络层使用的、**标识主机所在网络位置的逻辑地址**。	

IP 地址用于标识主机在网络中的逻辑位置，支持跨网络通信和路由选择，而 MAC 地址仅用于局域网内的链路层转发。

## 集线器 vs 交换机

### 集线器 —— 共享带宽

- **物理层设备**
- 所有端口共享一条带宽
- 半双工
- 数据 → 广播
- 集线器工作在 **物理层**，端口 **共享带宽**

------

### 以太网交换机 —— 独享带宽

- **数据链路层设备**
- 根据 MAC 表转发
- 全双工
- 每个端口 **独立冲突域**
- 交换机端口独享带宽

交换机各端口之间通信互不影响，**“各走各的路”**

## HDLC

- 面向比特的链路层协议
- 点到点 / 多点
- 可靠性强
- 主要用于专用链路

📌 特点：

- 结构严谨
- 不支持多网络层协议协商

## 点对点协议

简称为PPP。PPP由三部分组成

- 链路控制协议
- 网络控制协议
- 定义PPP帧格式

### PPP帧格式

![image-20251227160115981](D:\笔记\图片\image-20251227160115981.png)

- PPP帧通过标志字段`0x7E`作为帧定界符，标志帧的开始和结束。
- 第二个字节和第三个字节分别为PPP帧的地址域和控制域，固定为`FF`和`03`。
- 第四个字段为协议字段，占两个字节，指明了PPP帧的信息部分携带了哪一个协议的数据。
- 第五个字段是PPP帧的信息部分，携带具体的数据，长度可变，不超过1500B
- 下一个字段为`FCS`，存储CRC冗余校验码，判断是否发生比特错误。若出错，直接丢弃该帧。

### 透明传输

PPP帧支持两种填充方式实现透明传输

- 字节填充：用于异步传输。如果信息部分出现帧定界符一致的字符，则添加`0x7D`转义字符。
- 零比特填充：用于同步传输。除了首尾的帧定界符，只要发现有5个连续的1则立即在后面填入一个0

### LCP

链路控制协议，PPP的子协议。用于建立、配置LCP链路

### NCP

网络控制协议，PPP的子协议。为网络层协议建立和配置逻辑连接。

# 第四章：网络层

## 网络层的两大重要功能

1️⃣ **路由选择（Routing）**
 2️⃣ **分组转发（Forwarding）**

## 子网划分

子网划分 = 借主机位，扩展网络位

### 子网划分通用操作步骤

#### 已知：

- 一个网络地址
- 需要划分 **N 个子网**（或每个子网 M 台主机）

#### 解题步骤（考试模板）：

1️⃣ **确定需要借用的位数**

- 子网数 ≥ N
- 主机数 ≥ M + 2（网络地址 + 广播地址）

2️⃣ **修改子网掩码**

- 原掩码 + 借用位数

3️⃣ **计算子网地址**

- 步长 = 256 − 子网掩码对应位的值

4️⃣ **列出**

- 子网网络地址
- 子网广播地址
- 可用 IP 范围

## 路由聚合

将多个连续的网络地址合并为一个更大的网络进行路由通告。

- **减少路由表项**
- 提高路由效率

**网络聚合的核心条件（课上例题必考）**

- 多个网络：
  - **地址连续**
  - **前缀相同**
- 聚合后：
  - 网络位 **变短**
  - 主机位 **变多**

## 路由选择协议

路由选择协议用于路由器之间交换路由信息，生成和维护路由表。

### 路由信息协议

简称RIP。RIP属于应用层，使用UDP传输数据。RIP是一种基于距离向量的路由选择协议。

RIP通过跳数(也就是转发次数或称为距离)衡量到达目的地的距离，规定路由器到直连网络的距离为1，每经过一个路由器，距离加1。

RIP认为跳数最少的路由是最好的

RIP规定一条合法路径跳数不能超过15。超过15表示网络不可达。因此RIP只适用于小型自治系统。

RIP规定路由表中要包含三个关键字段:<目的网络N,距离d,下一跳路由器地址X>

RIP使用距离向量路由选择算法选择路由路径，距离向量的信息维护在路由表中。

<h4>RIP协议路由表建立过程</h4>
RIP 通过相邻节点交换整个路由表来更新路由信息。

1. 路由器刚启动时，只知道与自己相连的网络距离为1，会将与自己直接相连的网络记录在路由表中
2. RIP会向相邻路由器结点发送RIP响应报文，其中包含路由器完整的路由表信息
3. 路由器接收到RIP响应报文后，会修改报文中的路由表，简单来说就是将报文中的路由表项的跳数加1，并将下一条路由器地址记录为发送RIP报文的路由器。
4. 路由器会根据修改后的路由表更新自己的路由表：如果原本的路由表中没有包含目的网络，则将这个路由表项添加到自己的路由表；如果原本的路由表中已经包含目的网络，则比较两个路由表项的距离，保留距离较短的表项。
5. 此后，路由器会周期性的向相邻路由器发送RIP响应报文，路由器接收到路由表后，会按照上述规则修改自己的路由表

随着路由表迭代更新，所有结点最终都会知道到达本自治系统内任何网络的最短距离和下一跳路由器的地址，此时称RIP已收敛。

<h4>距离向量路由选择算法</h4>

- 每个结点维护一个距离向量(一组距离记录),记录本结点到所有目的结点的目前已知的最短路径长度。
- 初始每个结点仅知道与相邻结点之间的距离并建立距离向量，然后向相邻结点广播自己的距离向量。
- 当与相邻结点的距离发生变化或收到相邻结点发送的距离向量时，结点根据到相邻节点的距离以及相邻结点到目的结点的距离选择一条到目的结点距离最短的路径，计算到每个目的结点的最短距离，并更新自己的距离向量。
- 当距离向量发生变化时，结点向相邻结点广播自己的距离向量。同时，结点还会周期性广播距离向量。

### 开放最短路径优先协议

简称为OSPF协议。OSPF协议位于网路层，使用IP协议提供的服务传输数据。此时需要将IP数据报的首部协议字段修改为89。

- OSPF的PDU称为**OSPF分组**或**OSPF数据报**

OSPF协议定义了五种不同的OSPF分组，在OPSF分组的首部的`Type`字段中指定分组类型

| Type值 |             英文全称             |   英文缩写   |     中文译名     |
| :----: | :------------------------------: | :----------: | :--------------: |
|   1    |           Hello Packet           | Hello Packet |     问候分组     |
|   2    |   Database Description Packet    |  DD Packet   |  数据库描述分组  |
|   3    |    Link State Request Packet     |  LSR Packet  | 链路状态请求分组 |
|   4    |     Link State Update Packet     |  LSU Packet  | 链路状态更新分组 |
|   5    | Link State Acknowledgment Packet | LSAck Packet | 链路状态确认分组 |

OSPF协议基于链路状态路由选择算法(迪杰斯特拉算法)计算最佳转发路径。

为了使用迪杰斯特拉算法，首先需要让每个路由器建立整个网络的拓扑图(带权有向图)

<h4>OSPF协议建立网络拓扑图过程</h4>

OSPF 通过向整个区域广播链路状态信息，使所有路由器拥有相同的网络拓扑图。

1. 路由器在启动时，可以探测到与他直接相连的其他结点的信息，它会将所有相邻结点的信息通过单链表连接起来。
2. 之后，每一个结点会通过洪泛法向其他结点发送自己的链路状态信息。
3. 所有路由器结点就组成了一个带权有向图的邻接表。

### RIP 交换的内容：

- **路由表**
- 网络 → 跳数

### OSPF 交换的内容：

- **链路状态信息**
- 邻居、链路代价

# 第五章：传输层

传输层为应用进程之间提供端到端的通信服务。

## 传输层的两个协议

| 类型     | 特点             | 协议    |
| -------- | ---------------- | ------- |
| 面向连接 | 可靠、有序       | **TCP** |
| 无连接   | 不可靠、尽力而为 | **UDP** |

## 复用与分用

复用指不同的应用进程使用同一个传输层服务传输数据；分用指传输层能够把这些数据正确交付给目的应用进程

## IP地址+端口号

IP 地址：标识 **主机**

端口号：标识 **进程**

传输层通过 **IP 地址 + 端口号** 唯一标识一个通信端点。

## TCP

### 滑动窗口机制

滑动窗口是 TCP 实现可靠传输和流量控制的核心机制。

滑动窗口允许发送方在未收到确认前连续发送多个报文段。

### 流量控制

防止发送方发送过快，导致接收方缓存溢出。

基于 **滑动窗口**实现

接收方在 ACK 中通告：

- **接收窗口 rwnd**

## 拥塞控制

防止网络中出现过多数据，导致整体性能急剧下降。

**TCP 拥塞控制的四个阶段**

1️⃣ **慢开始：**cwnd值从1开始，每收到一个ACK，就让cwnd+1(当cwnd<ssthress时适用）
 2️⃣ **拥塞避免：**在一个RTT内，即使收到多个ACK，也只能让cwnd+1（当cwnd>ssthress时适用)
 3️⃣ **快重传**：当发送方收到三个确认号相同的冗余ACK时，立即重传对应报文段
 4️⃣ **快恢复：**一旦发生快重传，就将阈值和cwnd都设为当前cwnd的一半然后切换到为“拥塞避免算法’

### 超时重传机制

发送方在发送数据后，在规定时间(RTO)内未收到确认 ACK，就认为该数据丢失，并重新发送。

TCP 发送的每个报文段：

- 都有 **序号（Sequence Number）**

发送后：

- **不会立刻丢弃**
- 而是**保存在发送缓存中**

只有收到 ACK 后，发送缓存中的数据才会被删除。

**超时发生时，TCP 如何重传**

当 **RTO 到期** 且 **未收到 ACK**：

1. 发送方检查发送缓存
2. 找到 **最早未被确认的报文段**
3. **按原序号重新发送该报文段**

**超时时间 RTO **

RTO 是根据 RTT 的统计值动态计算出来的。RTT为往返时间。

**RTO 太短：**

- 频繁误判丢包
- 造成不必要重传
- 加重网络负担

**RTO 太长：**

- 丢包后等待时间过久
- 传输效率低
- 延迟增大
