# Hadoop

`Hadoop`是一套专门用于大数据处理的工具，内部包含多个组件。

**大数据处理的核心思想**

大数据难以处理的本质原因是数据量过大，机器性能具有瓶颈，因此对大数据进行高效处理的核心思想就是切分数据，将大数据切分成多份小数据，对小份数据进行存储，计算等一系列操作，不仅可以减轻处理每份数据的压力，还可以实现多份数据并行处理，提升效率。

<h3>为什么要使用<code>Hadoop</code>,数据存储在<code>Mysql</code>中进行处理不可以吗</h3>

Mysql不适用于大数据的处理(如TB，PB级别的数据)，最多只能用于`GB`级别数据的处理。当数据量达到TB级别后，且仍然再快速增长时，Mysql的处理非常低效，甚至可能直接卡死崩溃。所以，此时不再将数据存储在`MYSQL`中，而是使用`Hadoop`的一系列组件完成大数据的存储，处理等工作。

## `HDFS`

`Hadoop Distributed File System`是`Hadoop`生态系统中用于存储大数据的 **分布式文件系统**。

### 特点

**分布式存储**

HDFS把一个大文件拆分成多个 block(块),默认每块 128MB,不同块可以存储在不同存储节点上,且HDFS屏蔽了从不同存储节点读取数据的复杂性。

**面向大文件**

HDFS 设计主要针对 **TB/PB 级大文件**，小文件太多会导致 `NameNode` 元数据压力大。

**高可靠**

- 每块数据会复制多份(默认三份)，存储在不同节点上，以保证可靠性。

## `MapReduce`

`MapReduce` 是 `Hadoop` 处理大数据的核心计算模型和编程框架，用于实现分布式并行计算。

<h3>核心思想</h3>

`MapReduce`可以先将数据切分为多个分片，并对每个分片进行并行处理，它将对分片数据的处理抽象为主要两个阶段：

- `Map`(映射)：每个节点对自己的数据块执行 `Map` 函数,即对分片数据进行处理。
- `Reduce`(聚合)：执行`Reduce`函数将分片数据聚合为最终结果

## `Yarn`

`Yet Another Resource Negotiator`是 Hadoop 生态系统中的 **资源管理与调度系统**，它负责 **管理集群资源和调度作业**。

### 架构

`Yarn`将分配给作业的资源单位（CPU + 内存）抽象为一个容器(container)，并根据资源情况动态分配给作业。

# `Hive`

**Hive** 是 Hadoop 生态系统中的一个 **数据仓库工具**，主要作用是让你可以用 **类似 SQL 的语言** 查询和分析存储在 HDFS 上的海量数据。

# `Spark`

**Apache Spark** 是一个 **开源的大数据计算引擎**，专门用来处理海量数据，尤其擅长 **内存计算、并行计算和多场景分析**。

| 特性     | Spark                        | Hadoop MapReduce   |
| -------- | ---------------------------- | ------------------ |
| 计算方式 | 内存计算为主                 | 磁盘计算为主       |
| 速度     | 🚀 快                         | 🐢 慢               |
| 编程模型 | 灵活：Map/Filter/Join/SQL/ML | 固定：Map → Reduce |
| 实时处理 | ✅ 支持                       | ❌ 不支持           |
| 迭代计算 | ✅ 内存多轮迭代               | ❌ 每轮都读写磁盘   |
