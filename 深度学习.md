# Pytorch

> Pytorch是一个主要由C语言开发的轻量级深度学习框架，提供了丰富的Python接口便用户使用。

## Tensor

> 又名张量，Pytorch中最重要的数据结构，可以是一个数(标量)，一维数组(向量)，二维数组(如矩阵，黑白图像等)或更高维矩阵(如彩色图像，视频等)。
>
> Tensor可以使用GPU加速其运算。

### Tensor操作

> 为方便使用，Tensor的大部分操作同时支持torch模块方法和tensor对象成员方法这两类接口，即形如`torch.function`和`tensor.function`
>
> 在Tensor的各种操作中，函数名后带下划线`_`的函数被称为inplace操作，会修改调用者自身存储的数据。反之，不带下划线的函数会返回一个新的Tensor。

## autograd(自动微分)

> 在深度学习中，主要通过反向传播算法，计算梯度，再通过梯度下降来最小化损失函数，以此更新网络参数。Pytorch中的autograd模块实现了自动反向传播功能，optim模块实现了常见的梯度下降优化方法。

- 开启自动微分

  > 为Tensor设置requires__grad标识，表示需要求导

  ```python
  x = t.ones(2,2,requires_grad = True)
  # 或
  x.requires_grad_(True)
  ```

- 反向传播，计算梯度

  ```
  y = x.sum()
  y.backward()
  ```

  > backward()函数通常只能用于标量，如果对矩阵进行反向传播，可以先对矩阵求和再反向传播，结果不变
  >
  > 在进行一次反向传播后，计算图会被释放，所以不能对同一次前向传播计算得到的张量进行二次反向传播，除非设置backward()的参数`retain_graph=True`

- 获得梯度

  ```
  x.grad
  ```

  > 注意:
  >
  > - grad会在反向传播过程中累加，因此在每次进行反向传播前都需要将梯度清零

- 梯度清零

  ```
  x.grad.data.zero_()
  ```
  
- 禁止梯度计算

  > `with torch.no_grad()` 是 PyTorch 中用于**关闭自动求导（自动微分）功能的上下文管理器**

  ```
  with torch.no_grad():
  	参数更新
  ```

  > 在该代码块内，所有张量的操作都不会被记录到计算图中。通常用于模型推理（inference）阶段或手动更新参数时，避免无意义的梯度计算。
  
- 切断梯度追踪

  ```
  #detach函数返回一个新张量，它与原张量共享数据，但是不参与反向传播(即不计入计算图)
  x.detach()
  ```

  > 设置自动求导的张量转成`numpy`数组会报错(因为 `numpy` 不支持梯度追踪)

## 神经网络

> 基于autograd模块，Pytorch提供了torch.nn模块用来定义和运行神经网络，这是Pytorch中专门为神经网络设计的模块化接口。其中，nn.Module是nn模块中最重要的类，它包含了神经网络各层的定义以及前向传播(forward)方法

### 使用

1. 定义网络

   > 模型需要继承nn.Module,并实现它的forward方法。其中，网络中含有可学习参数的层，应该放到构造函数中。如果某一层(如RELU)不含有可学习参数，既可以把他放在构造函数中，也可也放在forward方法中。	

## 计算图(Computation Graph)

> 计算图是Pytorch等现代深度学习框架的核心，为反向传播 (  `Back Propogation ` )算法提供了计算基础。Pytorch在`autograd`模块中实现计算图相关功能。

- 静态计算图

  > 在神经网络定义阶段就会一次性把计算图完整构建好。

  - 特点
    - 训练时只是往图里“喂数据”，图的结构不会再变。
    - 编译优化空间大，执行速度快。
    - 调试不方便，写法受限（不能随意用 Python 控制流改变结构）。

- 动态计算图

  > 随着计算一步步构建计算图（边计算边建图）。Pytorch使用动态计算图

  - 特点
    - 每一次前向传播都会重新构建一张新的图。
    - 写法像普通 Python 代码一样，调试方便。
    - 优化空间比静态图小，执行速度有时稍慢。

### 前向传播（Forward Pass）

- **作用**：计算模型的输出，并在此过程中动态构建计算图并保存中间变量以计算梯度。
- **实现**：PyTorch 中通过 `Module` 的 `forward()` 方法定义前向计算逻辑。

------

### 反向传播（Backward Pass）

- **作用**：根据链式法则，沿计算图从输出层向输入层依次计算梯度。
- **触发方式**：调用 `loss.backward()` 自动完成梯度计算。

## 神经网络(nn)

> Pytorch提供了专门用于构建神经网络的nn模块，此模块基于autograd构建，提供了多种神经网络中常用的工具

### nn.Module

> nn.Module意为块，既指神经网络中的某一层，也可以指多个层组成的组件，又可以指整个神经网络。一个块可以由许多块组成。通常继承nn.Module实现自己的自定义块，需要实现构造函数与前向传播函数。块的任何子类都必须定义一个将其输入转换为输出的前向传播函数， 并且必须存储任何必需的参数。

```python
class Linear(nn.Module):
    def __init__(self,in_features,out_features):
        super().__init__() #必须要调用父类的构造方法，这会帮助完成Module的一些初始化工作
        #构造函数如果有自定义可学习参数需其封装成nn.Parameter
        self.W = nn.Parameter(t.randn(in_features,out_features))
        self.b = nn.Parameter(t.randn(out_features))

    def forward(self,x):
        x = x.mm(self.W)
        return x + self.b.expand_as(x)
    
layer = Linear(3,4)
y = layer(t.randn(2,3)) #调用时，直接通过实例化对象传入输入值，会自动调用forward()函数并做一些其他处理 
```

| reset_parameters() | 在nn.Module的子类中，会在init方法中自动调用这个方法，执行**默认的权重和偏置初始化**，不同的子类有不同的实现 |                                                              |
| state_dict()       | 返回当前块的参数字典，键为参数名，值为参数值                 | 如果当前块由多个块组成，会递归查询出所有参数，参数名为`索引.参数名` |
| named_parameters() | 返回一个迭代器，迭代器中保存的元素为元组，元组的第一个值为参数名，第二个为参数 |                                                              |
| eval()             | 切换模型到评估模式                                           | 会影响一些在训练和推理阶段行为不同的层(如Dropout)            |

### 前馈神经网络

> 每一层的输出都直接成为下一层的输入，这种神经网络被称为前馈神经网络(Feedforward Neural Network, FNN)。Pytorch提供了Sequential和ModuleList用以简化前馈神经网络的编写。它们是一个特殊的Module，可以包含几个子module。

#### nn.Sequential

> `nn.Sequential` 是 PyTorch 提供的**顺序容器**，用于按给定顺序堆叠多个块（`nn.Module`），实现 了简单的`forward()` 方法:它将列表中的每个块连接在一起，将每个块的输出作为下一个块的输入。

```python
#nn.Sequential已经实现了forward方法，会按照参数顺序逐层传播
net = nn.Sequential(
    nn.Linear(2, 4),
    nn.ReLU(),
    nn.Linear(4, 1)
)
#为各层起名后传入
net = nn.Sequential(OrderedDict([
    ('fc1', nn.Linear(2, 4)),
    ('relu', nn.ReLU()),
    ('fc2', nn.Linear(4, 1))
]))
#第三种使用方法
net = nn.Sequential()
net.add_module('conv',nn.Conv2d(3,3,3))
ent.add_module('batchnorm',nn.BatchNorm2d(3))
```

> nn.Sequential只适用于简单的前馈神经网络，因为他只实现了forward方法(顺序前向传播)

- 访问子module

  ```python
  #如果起名了，可以根据名字取出子module，也可以通过索引,如果块多层嵌套，可以通过链式法则连续调用取出子块
  net[0]
  net.conv
  ```

### 参数初始化

> 当网络层数增多时，一层层初始化参数会变得特别麻烦。因此，Pytorch提供了torch.nn.init模块方便初始化模型参数

|                           |                    |      |
| ------------------------- | ------------------ | ---- |
| normal_(param,mean,std)   |                    |      |
| zeros_(param)             |                    |      |
| constant_(param,constant) |                    |      |
| xavier_uniform_(param)    | 使用xavier均匀分布 |      |



- 默认初始化

  > 每一个由Pytorch提供的Module子类都实现了`reset_parameters()`方法，会在创建该层时使用默认定义的方法初始化参数

- Xavier初始化

  > Xavier 初始化（也叫 Glorot 初始化）是为了让每一层在前向传播和反向传播过程中，输出和梯度的方差在传播时保持一致。

  ![image-20250809170031153](D:\笔记\图片\image-20250809170031153.png)

  - Nin表示此层的输入特征数，Nout表示此层的输出特征数
  - 参数矩阵的方差应满足上式，因此Xavier初始化参数时通常会采用均值为0，方差满足上式的高斯分布或均匀分布
  
- 递归初始化

```python
def init_weights(m):
    if isinstance(m, nn.Linear):
        init.normal_(m.weight, mean=0, std=0.01)  # 用均值为0，标准差为0.01的正态分布初始化权重
        if m.bias is not None:
            init.constant_(m.bias, 0)             # 将偏置初始化为0

net.apply(init_weights)
```

- apply()方法继承自nn.Module,会递归的遍历该模块下的所有子模块，并对每个子模块都会调用传入的函数 `init_weights`，并把这个子模块作为参数 `m` 传进去。

### 优化器

> Pytorch将深度学习常用的优化方法封装在了torch.optim模块中。所有优化方法都继承自基类optim.Optimizer，并实现了自己的优化方法。

```
optimizer = optim.SGD (params=net.parameters(),lr=1)
output = net(input)
output.backward()
optimizer.step()
```

> 普通的Tensor不能直接放入优化器，必须使用nn.Parameter()包裹才可以(或者设置requires_grad为True),否则优化器会直接忽视它

#### SGD(随机梯度下降)

```python
optim.SGD(params,lr,momentum,weight_decay,dampening,nesterov,betas,eps,alpha)
#params可以接受两种类型的参数
#第一种:参数列表,此时通过关键字参数指定的超参数会应用到所有可学习参数中
optim.SGD(net.parameters(),lr=1)
#第二种:参数字典列表，关键字参数设置的是全局默认值，通过字典中的键指定本层的特定超参数。
optimizer = torch.optim.SGD([
    {'params': model.base.parameters(), 'lr': 0.01},        # 基础层
    {'params': model.classifier.parameters(), 'lr': 0.001, 'weight_decay': 0}  # 分类层
], lr=0.005, weight_decay=1e-4)  # 这是全局默认值
```

**`params`**

- 要优化的参数（模型的可训练参数），可以是 `model.parameters()` 或自定义的参数组（list of dict）。

**`lr`** *(float)*

- 学习率（Learning Rate）。默认值视优化器不同而定，例如 `SGD` 默认为 `1e-3`。

**`momentum`** *(float)*

- 动量因子（只在 `SGD` 等支持动量的优化器中使用）。

**`weight_decay`** *(float)*

- 权重衰退系数（L2 正则化系数），用于防止过拟合。

**`dampening`** *(float)*

- 动量的抑制因子（只在 `SGD` 中使用）。

**`nesterov`** *(bool)*

- 是否启用 Nesterov 动量（只在 `SGD` 中使用）。

**`betas`** *(Tuple[float, float])*

- 一阶与二阶矩估计的衰减率（只在 `Adam`、`AdamW`、`Adamax` 等优化器中使用）。
- 默认 `(0.9, 0.999)`。

**`eps`** *(float)*

- 防止除零的小常数（`Adam`、`RMSprop` 等中常用）。

**`alpha`** *(float)*

- 平滑常数（仅 `RMSprop` 使用）。

**`maximize`** *(bool)*

- 是否最大化目标函数（PyTorch 1.10+ 添加）。

**`foreach`** *(bool)*

- 是否使用更快的 `foreach` 实现（在部分优化器支持）。

**`capturable`** *(bool)*

- 是否允许在 CUDA 图（CUDA Graph）中捕获该优化器的操作（部分优化器支持）。

### DataLoader

> Dataset只负责数据集的抽象，每次调用`__getitem__`后，返回一个样本。但是实际训练时，每次需要处理一个batch的数据，且需要对数据进行打乱等操作。对此，Pytorch提供了DataLoader实现这些功能

```python
from torch.utils.data impoet DataLoader
DataLoader(dataset,batch_size=1,shuffle=False,sampler=None,batch_sampler=None,num_workers=0,collate_fn=None,pin_memory=False,drop_last=False,timeout=0,worder_init_fn=None,multiprocessing_context=None,geneaator=None,*,prefetch_factor=2,persistent_workers=False)
```

-  dataset：加载的数据集(Dataset对象）。
-  batch_size:一个batch的大小。
- shuffle：是否将数据打乱。
- sampler：样本抽样，后续会详细介绍。
- batch_sampler:与sampler 类似，一次返回一个batch的索引 （该参数与 batch _size、shuffle、sampler、drop_Iast 不兼容)。
- num_workers：使用多进程加载的进程数，0代表不使用多进程。
- collate_fn：如何将多个样本数据拼接成一个 batch，一般使用默认的拼接方式即可。
- pin_memory：是否将数据保存在pin memory区，将pin memory 区的数据转移到GPU上速度更快。
- drop_last: Dataset 中的数据个数可能不是 batch_size 的整数倍，若drop_last 为True，则将多出来的不足一个batch 的数据丢弃。
-  timeout：进程读取数据的最大时间，若超时，则丢弃数据。
- worker_init_fn;每个worker的初始化函数。
- multiprocessing_context:指定多进程的上下文环境，可以指定为已经启动的进程
- generator：多进程使用的生成器。
- persistent_workers:在加载完一个epoch的数据后，是否保留当前的worker。
- prefetch_factor：每个worker 预先加载的样本数。

> DataLoader是一个可迭代对象，可以多次迭代它，如果设置shuffle为True，则迭代一轮后会自动打乱数据
>
> 不要在DataLoader中对数据进行处理，而是在Dataset中
>
> DataLoader可以接受任何可索引的可迭代对象，不只是Dataset

## 神经网络的通用训练步骤

1. 定义一个包含可学习参数的神经网络。
2. 加载用于训练该网络的数据集。
3. 进行前向传播，得到网络的输出结果，计算损失（网络输出结果与正确结果的差距）。
4. 进行反向传播，更新网络参数。
5. 保存网络模型。

### 卷积神经网络

> Convolutional Netural Network，简称CNN。是一种特殊的神经网络，适合处理图像，视频，音频等"具有空间结构"的数据。

- 特点

  > 在普通前馈神经网络（即多层感知机 MLP）的基础上，加入了 **卷积层（Convolutional Layer）** 和 **池化层（Pooling Layer）** 等结构，形成的一个更加适合处理图像任务的网络结构。

- 与传统神经网络对比

  | 特点         | 普通神经网络（MLP） | 卷积神经网络（CNN）            |
  | ------------ | ------------------- | ------------------------------ |
  | 输入数据结构 | 一维向量            | 通常是二维或三维结构（如图像） |
  | 参数量       | 多，容易过拟合      | 少，具有**参数共享**特性       |
  | 局部特征提取 | 没有                | 有（卷积核提取局部信息）       |
  | 空间不变性   | 不具备              | 具备（池化层带来平移不变性）   |

- 组成

  1. **卷积层（Convolutional Layer）**
     - 使用**卷积核（过滤器）**滑动窗口提取局部特征。
     - 每个卷积核只关注输入的一小部分区域（局部感受野）。

  2. **激活函数（如 ReLU）**
     - 引入非线性，使得网络能够拟合复杂函数。

  3. **池化层（Pooling Layer）**

     - 对特征图进行降采样，减少计算量，提取主要特征。

     - 常用方法如最大池化（MaxPooling）或平均池化（AveragePooling）。

  4. **全连接层（Fully Connected Layer）**
     - 类似普通神经网络，用于最后的分类或回归任务。

## LeNet

> 最早的卷积神经网络，LeNet共有7层，它的输人图像的大小为32×32，共经过两个卷积层、两次下采样操作以及三个全连接层，得到最终的10维输出。
