# 二分查找(Binary Search)

> 又称二分法，是从一个有序数组中找到一个`target`值的搜索算法

- 核心思想

  > 由于数组是有序的，可以设定一个区间，每次比对区间中间的值并根据值与target的相对大小缩小区间，每次可以缩小一半的区间，所以称为二分法。

## 实现

> 根据标识区间范围的两个数为左闭右开和左闭右闭，可以将二分法分为两种写法

- 左闭右闭

  ```java
  public static int BinarySearch(int[] array,int target) {
          int indexMin = 0;
          int indexMax = array.length - 1;
          while (indexMin <= indexMax) {
              int indexTemp = (indexMax + indexMin) / 2;
              if (array[indexTemp] > target) {
                  indexMax = indexTemp - 1;
              } else if (array[indexTemp] == target) {
                  return indexTemp;
              } else if (array[indexTemp] < target) {
                  indexMin = indexTemp + 1;
              }
          }
          return -1;
      }
  ```

- 左闭右开

  ```
  public static int BinarySearch(int[] array,int target) {
          int indexMin = 0;
          int indexMax = array.length;
          while (indexMin < indexMax) {
              int indexTemp = (indexMax + indexMin) / 2;
              if (array[indexTemp] > target) {
                  indexMax = indexTemp;
              } else if (array[indexTemp] == target) {
                  return indexTemp;
              } else if (array[indexTemp] < target) {
                  indexMin = indexTemp + 1;
              }
          }
          return -1;
      }
  ```

> 注意:
>
> - 对于循环判断条件和边界值的更新，要保持与初始区间一致的设定以及合法性，如果初始设定区间为左闭右闭则后续更新区间时也必须是左闭右闭

# 删除数组元素

> 给定一个数组以及一个特定元素，删除数组中的特定元素。

- 核心思路

  > 本质是将需要被删除的特定元素覆盖，即将后续元素向前移，覆盖需要被删除的元素，可以使用双层for循环暴力破解，也可以使用双指针

## 实现

- 双层for循环

```
 int size = nums.length;
        for (int i = 0; i < size; i++) {
            if (nums[i] == target) {
                for (int i1 = i; i1 < size - 1; i1++) {
                    nums[i1] = nums[i1 + 1];
                }
                i--;
                size--;
            }
        }
        return size;
```

- 双指针

  > 时间复杂度为O(n),在这里，快指针用于获得删除后新数组中的元素，慢指针是新数组中元素的位置，两者相互配合，将需要的元素移入新的数组中。

  ```java
  public static int elementDelete (int[] nums, int val) {
          int slowPtr = 0;
          int fastPtr = 0;
          for (; fastPtr < nums.length; fastPtr++) {
          //如果不等于val，表示这个元素应该在新数组中，将快指针指向的元素移到慢指针指向的位置并更新慢指针。
              if (nums[fastPtr] != val) {
                  nums[slowPtr] = nums[fastPtr];
                  slowPtr++;
              }
          }
          return slowPtr;
      }
  ```

# 最小子数组

> 给定一个含有 `n` 个正整数的数组和一个正整数 `target` **。**
>
> 找出该数组中满足其总和大于等于 `target` 的长度最小的 **子数组** `[numsl, numsl+1, ..., numsr-1, numsr]` ，并返回其长度**。**如果不存在符合条件的子数组，返回 `0` 。

- 核心思想

  > 可以使用一个滑动窗口扫描数组，并根据滑动窗口所框住子数组的和来动态的移动和改变滑动窗口的大小和位置，本质仍然是双指针。

```java
public int minSubArrayLen(int target, int[] nums) {
        int lowPtr = 0;
        int highPtr = 0;
        int sum = 0;
        int result = 0;
       for (;highPtr < nums.length; highPtr++) {
           sum += nums[highPtr];
           while (sum >= target) {
               int subLength = highPtr - lowPtr + 1;
               result = result != 0 ? Math.min(subLength,result) : subLength;
               sum -= nums[lowPtr];
               lowPtr++;

           }
       }
       return result;
    }
```

# 螺旋矩阵

> 一个正整数 `n` ，生成一个包含 `1` 到 `n2` 所有元素，且元素按顺时针顺序螺旋排列的 `n x n` 正方形矩阵 `matrix` 。

- 核心思想

  > 将二维数组看成一个矩阵，以每螺旋一圈为循环，在这个循环中又以边为单位进行处理。通过定义矩阵坐标，以螺旋的形式更新矩阵坐标并赋值。

```java
 public int[][] generateMatrix(int n) {
        int startX = 0;
        int startY = 0;
        int loopCount = 1;
        int count = 1;
        int[][] nums = new int[n][n];

        while (loopCount <= n / 2) {
            int j = startY;
            int i = startX;
            //上边
            for (; j < n -loopCount;j++) {
                nums[i][j] = count;
                count++;
            }
            //右边
            for (;i < n - loopCount;i++) {
                nums[i][j] = count;
                count++;
            }
            //下边
            for (;j > startY;j--) {
                nums[i][j] = count;
                count++;
            }
            //左边
            for (;i > startX;i--) {
                nums[i][j] = count;
                count++;
            }
            loopCount++;
            startX++;
            startY++;
        }
        if (n % 2 == 1) {
            nums[startX][startY] = n * n;
        }
        return nums;
    }
```

> 注意:
>
> - 要以相同的方式处理每一边，即以相同的方式看待相邻边共同节点的归属权。
> - 如果螺线矩阵的边长为基数，此时最中心有个元素无法构成一圈，因此单独判断进行赋值

## 解数独

数独的解法需 **遵循如下规则**：

1. 数字 `1-9` 在每一行只能出现一次。
2. 数字 `1-9` 在每一列只能出现一次。
3. 数字 `1-9` 在每一个以粗实线分隔的 `3x3` 宫内只能出现一次。

```java
    public void solveSudoku(char[][] board) {
        backtracking(board);
    }
//检查遇到的第一个空格，并向里面填数
    private boolean backtracking(char[][] board) {
        for (int row = 0; row < 9; row++) {
            for (int col = 0; col < 9; col++) {
                if (board[row][col] != '.') continue;
                for (int num = 1; num <=9;num++) {
                    if (!isValid(board,row,col,num)) {
                        continue;
                    }
                    board[row][col] = (char) ('0' + num);
                    if (backtracking(board)) return true;
                    board[row][col] = '.';
                }
                return false;
            }
        }
        return true;
    }

    public boolean isValid(char[][] board,int row ,int column,int num) {
        //判断数字在行上是否有效
        for (int i = 0; i < board.length; i++) {
            if (board[row][i] == (char)('0' + num)) {
                return false;
            }
        }
        //判断数字在列上是否有效
        for (int i = 0; i < board.length; i++) {
            if (board[i][column] == (char)('0' + num)) {
                return false;
            }
        }
        //判断数字在九宫格内是否有效
        int i = row / 3;
        int j = column / 3;
        for (int k = 0; k <3; k++) {
            for (int l = 0; l < 3; l++) {
                if (board[i * 3 + k][j * 3 + l] == (char)('0' + num)) {
                    return false;
                }
            }
        }
        return true;
    }
```

# 滑动窗口

**概念**：在数组或字符串上维护一个窗口（通常用两个指针 `left` 和 `right` 表示窗口的左右边界），窗口内的数据满足一定条件，窗口不断“滑动”以完成问题求解。

**目的**：减少重复计算，实现 **O(n)** 或接近 O(n) 的时间复杂度。

**核心操作**：

1. **扩大窗口**：移动右指针，包含更多元素，直到不满足条件。
2. **收缩窗口**：移动左指针，剔除部分元素，使窗口恢复满足条件。
3. **更新结果**：每次窗口调整后，根据题意更新答案（如长度、和、最大值等）。

# LRU算法

> `Least Recently Used`,最近最少使用。一种资源淘汰算法。

**核心思想**

> 哪个资源最近最久没被使用，就把他淘汰。

**应用场景**

- 缓存

**实现思路**

- 通过双向链表维护资源的最近使用情况，谁最近使用过，就将他插入链表头。
- 这样链表尾部就是天然的最近最少使用的数据

## LRU缓存

> Least Recently Used Cache（最近最少使用缓存），是一种根据`LRU`算法产生的缓存淘汰策略，用于管理有限容量的缓存，保证最常用的数据能够保留在缓存中，而不常用的数据被淘汰。

**核心思想**

​		如果缓存被占满，则淘汰**最近最少使用**的数据。

### 实现

- 通常使用双向链表与哈希表来实现LRU缓存
- 哈希表存储每一个缓存项的`key`,以及由`key`,`value`标识的双向链表节点
- 通过哈希表查询与存储缓存项，通过双向链表维护缓存项的最近使用情况，如果最近使用过，则其靠近头节点，否则则靠近尾节点。
- 哈希表的查询与写入时间复杂度均为O(1),双向链表移动节点位置以及删除节点的时间复杂度也为O(1)

```java
public class LRUCache {
    private HashMap<Integer,Node> map;
    private Node head;
    private Node tail;
    private int capacity;
    public LRUCache(int capacity) {
        map = new HashMap<Integer,Node>(capacity);
        this.capacity = capacity;
        head = new Node(0,0);
        tail = new Node(0,0);
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        if (map.containsKey(key)) {
            Node node = map.get(key);
            node.prev.next = node.next;
            node.next.prev = node.prev;
            Node temp = head.next;
            head.next = node;
            node.next = temp;
            node.prev = head;
            temp.prev = node;
            return node.getValue();
        }
        return -1;
    }

    public void put(int key, int value) {
        if (map.containsKey(key)) {
            Node node = new Node(key,value);
            Node removeNode = map.remove(key);
            removeNode.prev.next = removeNode.next;
            removeNode.next.prev = removeNode.prev;
            map.put(key,node);
            Node temp = head.next;
            head.next = node;
            node.next = temp;
            node.prev = head;
            temp.prev = node;
            return;
        }
        if (map.size() == capacity) {
            int removeKey = tail.prev.getKey();
            map.remove(removeKey);
            tail.prev.prev.next = tail;
            tail.prev = tail.prev.prev;
        }
        Node node =new Node(key,value);
        map.put(key,node);
        Node temp = head.next;
        head.next = node;
        node.next = temp;
        node.prev = head;
        temp.prev = node;
    }
}
public class Node {
    private int key;
    private int value;
    public Node next;
    public Node prev;
    public Node(int key, int value) {
        this.key =key;
        this.value = value;
    }
    public int getKey() {
        return key;
    }
    public int getValue() {
        return value;
    }
}
```

# LFU算法

> `Least Frequently Used`,最近最不常用。一种资源淘汰算法，根据数据访问次数来淘汰数据，当访问次数一致时，删除最久未使用的数据。

**核心思想**

如果数据过去被访问过多次，那么将来被访问的频率也更高；反之亦然。

# 协同过滤

协同过滤(Collaborative Filtering, CF)是推荐系统里最经典的一种实现方法，它的核心思想是利用历史行为数据找相似性。协同过滤可分为基于用户的协同过滤与基于物品的协同过滤

**基于用户的协同过滤**

基于历史行为寻找与用户相似的人，推荐与用户兴趣相似的人接触过而用户没有接触过的东西

**基于物品的协同过滤**

基于历史行为寻找相似的物品，推荐与用户感兴趣物品相似的物品

## ✅ 优点

- 不需要理解内容（简单）
- 推荐效果好（基于真实行为）
- 可扩展性强

## ❌ 缺点

1. **冷启动问题**
   - 新用户 → 没数据
   - 新物品 → 没人用
2. **数据稀疏**
   - 用户行为少 → 不好算相似度
3. **热门偏置**
   - 热门内容更容易被推荐（马太效应）

<h3>获得大于等于指定整数cap的最小2的幂</h3>

**位扩散法**

```java
    public static int tableSizeFor(int cap) {
        //处理 cap 本身就是 2 的幂的情况
        int n = cap - 1;
        //把最高位的 1 后面的所有位都变成 1
        n |= n >>> 1;
        n |= n >>> 2;
        n |= n >>> 4;
        n |= n >>> 8;
        n |= n >>> 16;
        return n > 0 ? n + 1 : 1;
    }
```

- 将 `cap-1` 的最高位 1 向右扩散成全 1，再加 1，从而在 O(1) 时间内计算出 **大于等于 cap 的最小 2 的幂容量**
