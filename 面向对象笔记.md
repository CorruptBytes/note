# 类与对象

类（Class）是对象的模板，它定义了一组具有相同属性和行为的对象的公共特征。

对象（Object）是类的实例，具有类定义的属性和行为，是程序运行中的具体实体。

**类的定义**

类的属性由类的成员变量定义，类的行为由类的成员方法定义。

```java
public class Person {
    //成员变量
    public String name; 
    //成员方法
    public void sayHello() { 
        System.out.println("Hello, I'm " + name);
    }
}
```

- 一个Java文件中可以平行的定义多个类，但只能有一个类被public修饰，且这个类必须与Java文件同名。

## 类的组成

> 从面向对象思想的角度，类由属性(事物的特征)与行为(事物能执行的操作)组成；
> 从Java的语法层面看，类的完整组成包括五大成员:①属性②方法③构造方法④代码块⑤内部类

- 代码块(不包括静态代码块)中的指令会被合并到构造方法中，并且会放在构造方法原有指令之上执行。因此每次创建对象都会执行代码块
- 静态代码块的指令会被整理到`clinit`方法中，这个方法会在类初始化时被调用。因此静态代码块只会在类初始化时执行一次。

## 类和对象的关系

- 类是对象的模板，对象是类的实例。

- 类定义了对象的属性和行为，而对象则是在程序运行时根据类创建出来的实体。
- 同一个类可以创建多个对象，每个对象拥有属于自己的属性值。

## 对象的创建与使用

> 类是创建对象的模板，通过类的构造方法可以创建对象

```java
//创建对象
类名 对象名 = new 类名(参数);
//调用对象成员
对象名.成员变量
对象名.成员方法();
```

> 注意:
>
> - 使用new关键字调用类的构造方法来创建实例

## 构造方法

> 构造方法是类中一种特殊方法，用于创建对象并完成对象的初始化。

```java
//定义
修饰符 类名 (参数) {

}
//调用
类名 对象名 = new 类名(参数);
```

> 注意:
>
> - 如果类中没有定义任何构造方法，编译器会自动添加一个无参数构造方法；如果定义了任何一个构造方法，编译器不会提供默认构造方法
> - 构造方法必须通过new关键字调用。
> - 构造方法支持重载，可以定义多个构造方法，编译器会根据传入的参数选择正确的构造方法创建对象

## 访问器(getter)，更改器(setter)

> 访问器与更改器是类中特殊的成员方法，用于访问与修改类中的私有属性。

```java
// 访问器（Getter）：用于读取属性值
public 数据类型 get属性名() {
    return 属性名;
}

// 修改器（Setter）：用于修改属性值
public void set属性名(数据类型 属性名) {
    this.属性名 = 属性名;
}
```

> 注意:
>
> - getter与setter以get/set开头，后跟属性名首字母大写。
> - 通过将成员变量设为私有（`private`），避免外部类对其直接访问和修改，转而通过公开的访问器（`getter`）和修改器（`setter`）提供安全、受控的访问方式。这是封装的核心思想之一。

## 静态成员

> 通过static关键字，可以定义类的静态成员，静态成员属于类，而不属于由类创建的对象。

- 静态变量

  > 使用static关键字声明的变量，属于类而不属于具体的对象

  - 特点

    - 共享

      > 所有该类的实例共享同一个静态变量

    - 初始化

      > 静态变量在类被加载时初始化，只会进行一次内存分配

    - 访问方式

      > 静态变量可以直接通过类名访问，也可以通过实力访问。

- 静态方法

  > 类中使用static关键字声明的方法，属于类，而不是任何具体的对象

  - 特点

    - 无实例依赖

      > 静态方法可以在没有创建类实例的情况下通过类名直接访问，静态方法中，不饿能直接访问非静态的成员变量或方法，因为静态方法发没有上下文的实例

    - 访问静态成员

      > 静态方法中可以直接调用其他静态变量和静态方法

    - 多态

      > 静态方法无法被重写，但可以被隐藏。

## 内部类

​		将类A定义在另一个类B里面，则类A被称为内部类，类B被称为外部类

> 注意:
>
> - 内部类必须依赖于外部类才有意义

**外部类可以直接访问内部类的任何字段和方法（包括 private、默认、protected、public）**，**内部类也可以直接访问外部类的任何字段和方法（包括 private）**

### 成员内部类

​		将内部类定义在外部类的成员位置，且无static修饰的内部类称为成员内部类

```java
//定义:

//使用:

//创建内部类对象:
方式1: 在外部类中编写方法，对外提供内部类对象
方式2: 外部类名.内部类名 对象名 = new 外部类类名().new 内部类类名()
```

- 特点
  - 属于外部类实例对象的
  - 成员内部类可以被权限修饰符修饰，如果被private修饰，则外界无法访问内部类
  - JDK16之前不能在成员内部类中定义静态变量，JDK16后可以
  - 创建内部类对象时，对象中有一个隐含的Outer.this引用记录外部类对象的地址值。
  - 内部类可以直接访问外部类的成员，包括私有
  - 创建成员内部类之前，必须先创建外部类对象

### 静态内部类

​		被static修饰符修饰的成员内部类被称为静态内部类，是特殊的成员内部类，静态内部类中可以定义静态成员，也可以有非静态成员

```java
创建: 外部类名.内部类名 对象名 = new 外部类名.内部类名()
调用静态方法:外部类名.内部类名.方法名()
调用非静态方法: 先创建对象，用对象调用
```

- 特点
  - 静态内部类不依赖于外部类的实例对象
  - 静态内部类只能访问外部类的静态成员，想要访问外部类的其他成员，需要通过实例化外部类访问
  - 成员内部类在外部类实例化后才能实例化，而静态内部类可以独立实例化

### 局部内部类

> 将内部类定义在方法中，被称为局部内部类

```
public void show () {
	class Inner {
	
	}
}
```

- 特点

  - 局部内部类类似方法中的局部变量，无法用权限修饰符修饰

  - 局部内部类可以直接访问外部类的成员，也可以访问方法内的局部变量
  - 外界无法直接使用，需要在方法内创建对象并使用

### 匿名内部类

> 隐藏了名字的内部类，可以写在成员位置，也可以写在局部位置

```java
格式: new 类名或接口名(参数) {
	重写方法
}
```

- 匿名内部类的创建包含三部分:①继承/实现②方法重写③创建对象，本质是创建了一个子类对象并重写了其方法。
- 当一个方法的参数是接口或类时，而这个接口或类只会使用一次，此时就可以使用匿名内部类简化代码
- 可以向父类构造方法传参从而手动初始化继承自父类的一些属性
- `{}`中可以添加`{}`实例初始化代码块，会在构造方法执行完毕后执行。

### 非静态内部类访问外部类成员

​		编译器在生成字节码文件时会为非静态内部类维护一个指向外部类实例的引用。这个引用使得非静态内部类能够访问外部类的成员。编译器会在生成非静态内部类的构造方法时，将外部类实例作为参数传入，并在内部类的实例化过程中建立外部类实例与内部类实例之间的联系，从而实现直接访问外部方法的功能

## 枚举类

> 枚举类是一种通过`enum`关键字声明的特殊的类,枚举类经编译器编译后本质仍然是通过`class`声明的类，只是编译器对其进行了特殊处理。

**应用场景**

- 用于信息分类与标志,代替常量类

**定义枚举类**

```java
public enum 枚举类名 {
    //如果直接声明枚举值，则会默认使用无参构造初始化枚举值,否则会使用对应参数的构造方法声明枚举值
    枚举值1,枚举值2,枚举值3(val1,val2)···;
    Type field; //声明成员字段
}
```

- 枚举值必须全部声明在枚举类的最顶部
- 枚举类与普通类一致，如果显式提供了构造方法，则不会自动提供默认的构造方法

**枚举类反编译文件**

```java
public final class com.study.MyEnum extends java.lang.Enum<com.study.MyEnum> {
  public static final com.study.MyEnum X;
  public static final com.study.MyEnum Y;
  public static final com.study.MyEnum Z;
  public static com.study.MyEnum[] values();
  public static com.study.MyEnum valueOf(java.lang.String);
  static {};
}
```

- 枚举值均是由`public static final`修饰的类型为枚举类的常量，编译器会在静态代码块中自动调用构造方法初始化枚举值
- 如果未定义构造方法，编译器会自动生成一个带两个参数的构造方法`Enum(String name, int ordinal)`,在此构造方法中调用父类构造方法初始化`name`与`ordinal`属性。枚举常量初始化时会自动传入枚举名和序号（从 0 开始）
- 如果自定义的构造方法未定义`name`与`ordinal`参数，编译器会在构造方法前自动添加 `(String name, int ordinal)` 参数。初始化枚举常量时，自动传入枚举名和序号。
- 如果自定义构造方法带有`name`与`ordinal`，此时必须在初始化枚举值时手动传入参数值。且构造方法第一行必须显式调用父类 `Enum` 的构造方法，否则报错
- `name`与`ordinal`为`Enum`父类继承来的属性
- 所有枚举类均自动继承`java.lang.Enum`类
- 编译器会为枚举类自动生成两个静态方法`values`,`valueof(String name)`，`values()`返回一个封装了枚举值的数组,`valueof()`根据枚举值名字获得枚举值。
- 将枚举类声明为内部类时，会隐式被static修饰变为静态内部类

**Enum父类为枚举类提供了一些方法，可以通过枚举值调用他们**

|                  |                        |      |
| ---------------- | ---------------------- | ---- |
| `Enum.name()`    | 返回枚举值的名字       |      |
| `Enum.ordinal()` | 返回调用者枚举值的索引 |      |



##  数据载体类

> 可通过`record`语法糖定义数据载体类，这是一种特殊的类，只用于保存数据，不包含任何逻辑

**特点**

- 不能被继承
- 一旦创建，字段不可修改(不可变对象)
- 可添加自定义方法或静态成员
- 可实现接口，但不能继承其他类

**定义数据载体类**

```java
public record Person(String name, int age) {}
```

​	**反编译文件**

```java
public final class Person extends java.lang.Record {
    private final String name;
    private final int age;

    // 自动生成构造器
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 自动生成访问器方法（不是 getter）
    public String name() { return name; }
    public int age() { return age; }

    // 自动生成 equals、hashCode、toString 方法
    @Override
    public boolean equals(Object o) { ... }

    @Override
    public int hashCode() { ... }

    @Override
    public String toString() { ... }
}

```

# 面向对象

面向对象是一种编程范式，它的核心思想是将现实世界中的事物抽象为对象，对象具有属性（字段）和行为（方法）。**以对象为中心**，通过对象之间的交互来实现程序的功能。

面向对象的三大特性包括：**封装**、**继承**和**多态**。

## 封装

> 封装指将对象的属性（数据）和方法（操作）打包在一个整体中，并通过权限控制隐藏其内部实现细节，仅对外暴露必要的接口。

- 作用
  - 数据保护
  - 简化使用
  - 增强模块独立性

### 封装的实现

> Java中通过类来组织数据与操作，通过将成员变量设为私有（`private`），避免外部类对其直接访问和修改，转而通过公开的访问器（`getter`）和修改器（`setter`）提供安全、受控的访问方式。

```java
class Student {
    //成员变量
    String name;
    private int age;

    //提供get/set方法
    public void setAge(int a) {
        if(a<0 || a>120) {
            System.out.println("你给的年龄有误");
        } else {
            age = a;
        }
    }

    public int getAge() {
        return age;
    }

    //成员方法
    public void show() {
        System.out.println(name + "," + age);
    }
}
```

## 继承

> 继承是一种可以使得子类共享父类属性和方法的机制，一个类（子类）可以继承另一个类（父类）的属性和方法，从而实现代码的重用和扩展。

- 作用
  - 代码复用
  - 建立类与类之间的层次关系使得结构更加清晰

## 多态

> 多态指同一个行为（方法调用）在不同对象上的不同表现形式。分为编译时多态(重载)和运行时多态(重写)。

- 作用
  - 解耦，增强了程序的灵活性与扩展性

### 多态的实现

- 存在继承或实现关系
- 方法重写(子类重写父类或接口中的方法)
- 向上转型(父类或接口的引用指向子类对象)

### 多态的体现

- 方法重载

  > 在同一个类中，存在多个同名方法，它们具有不同的参数列表(参数类型，数量或顺序不同)。虽然方法名相同，但根据传入的参数不同，编译器会在编译阶段确定调用哪个方法。又称为编译时多态

- 方法重写

  > 指子类能够提供对父类中同名方法的具体实现。在运行时，通过父类的引用调用方法，JVM会根据调用者的实际类型确定调用哪个版本的方法。称为运行时多态。这是实现多态的主要方式

- 接口与实现

  > 多个类可以实现同一个接口并且用接口类型的引用来调用这些类的具体实现。这也属于运行时多态

  |        多态类型         |  多态形式  | 判定时机 |                    简要说明                    |
  | :---------------------: | :--------: | :------: | :--------------------------------------------: |
  | 方法重载（Overloading） | 编译时多态 | 编译阶段 |        同一个类中，方法名相同，参数不同        |
  | 方法重写（Overriding）  | 运行时多态 |  运行时  |  子类重写父类方法，父类引用调用时执行子类方法  |
  |       接口与实现        | 运行时多态 |  运行时  | 接口引用指向实现类对象，调用的是具体实现的方法 |

  

### 向上转型

> 向上转型就是用父类或接口的引用指向子类对象。通过这种方式，可以在运行时期调用不同的子类实现。这是多态实现的前提之一。

- 特点

  - 无需强制转换，自动发生

    ```
    Animal dog = new Dog();
    ```

  - 通过父类或接口引用只能访问父类或接口中定义的成员，不能访问子类特有成员

  - 方法调用时会根据不同的对象调用不同的实现

### 向下转型

> 向下转型是将父类引用转回子类类型，用于还原对象的真实类型，以调用子类特有的方法。但在执行前需要确认引用实际指向的对象类型，否则会出现ClassCastException异常

## 抽象类

> 抽象类是使用abstract关键字声明的类。一般作为基类，被其它类继承和扩展使用。包含抽象方法的类就是抽象类,抽象类中不一定有抽象方法，有抽象方法的类一定是抽象类.

```
public abstract test {

}
```

> 注意:
>
> - 抽象类中可以包含实例变量与静态变量
> - 

- 特点

  - 抽象类不能被实例化，只能继承；不过抽象类可以具有构造方法(帮助子类赋值)
  - 抽象类中的方法可以没有方法体，这些方法叫做抽象方法，需要用abstract关键字声明
  - 抽象类的子类，要么是抽象类，要么重写抽象类中的所有抽象方法

- 抽象方法

  > 将共性的行为抽取到父类之后，由于每个子类的具体实现是不同的，所以父类无法确定方法体，此时可以不实现方法，将其定义为抽象方法。抽象方法就是没有方法体的方法

  ```java
  public abstract void test();
  ```

## 接口

> 接口定义对类行为的规范

```java
public interface flyable {
  	// 接口常量
    int VERSION = 1; // public static final

    // 抽象方法
    void show();
}
```

 - 在Java8以前，接口中只能定义常量与抽象方法。
 - 接口与类的关系是实现，任何类实现接口，就必须提供接口中所有抽象方法的具体实现（除非该类是抽象类）。
 - 一个类可以实现多个接口
- 接口中的抽象方法默认且只能由`public abstract`修饰
 - 接口中的成员变量默认且只能由` public static final`修饰，也就是说接口中定义的成员变量实际上是一个常量。
 - 常量必须要给初始值。常量命名规范建议字母全部大写，多个单词用下划线连接。

**java 8 后接口中可以有默认方法和静态方法，Java9后可以有私有方法，用于在接口中为默认方法或其他私有方法提供辅助功能**

```java
interface Example {
    // 接口常量
    int VERSION = 1; // public static final

    // 抽象方法
    void show();

    // 默认方法（非抽象）
    default void log(String msg) {
        System.out.println("LOG: " + msg);
    }

    // 静态方法
    static void info() {
        System.out.println("Interface static info");
    }
}
```

- 特点

  - 接口不可以有构造函数，不能被实例化

- 实现接口

  > 类与接口的关系为实现关系，即类实现接口，该类可以称为接口的实现类，也可以称为接口的子类。实现的动作类似继承，格式相仿，只是关键字不同，实现使用 implement关键字。

  ```
  public class Bird implements flyable {
  
  }
  ```

  > 注意:
  >
  > - 实现类要么重写接口中的所有抽象方法，要么声明为抽象类。

- 接口的继承

  > Java中，接口与接口之间可以存在继承关系，而且接口之间可以多继承

### 密封接口

JDK17引入，只允许`permits`关键字后声明的类实现此接口。如果其他类实现此接口，编译时会报错。

```
public sealed interface CommandResult permits A, B, C {}
```

- 密封接口的实现类必须被`final`,`sealed`,`non-sealed`其中一个修饰

  | 关键字       | 含义                   |
  | ------------ | ---------------------- |
  | `final`      | 不允许再被继承         |
  | `sealed`     | 继续限制它的子类       |
  | `non-sealed` | 放开限制，允许任意继承 |
