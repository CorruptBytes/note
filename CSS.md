# 概述

> Cascading Style Sheets,层叠样式表,也叫CSS样式表或级联样式表。
>
> 与HTML相同，CSS也属于标记语言
>
> CSS主要用于设置HTML页面的文本内容样式(字体，字号，对齐方式),图片外形(宽高，边框样式，边距)以及版面的布局和外观显示样式。

## 语法

> CSS语法主要有两部分组成:选择器以及选择器中的样式声明。

```css
h1 {color:red;  font-size:25px;}
```

> 注意:
>
> - 选择器用于指定被附加CSS样式的HTML元素，后跟花括号包含多条样式声明
> - 样式声明以冒号键值对的方式出现，多条声明之间用`;`隔开

## 引入方式

### 行内样式表(行内式)

 > 行内样式表(内嵌样式表)将样式声明在元素标签的style属性中，修改简单样式时可用

 ```css
 <div style="color: red; font-size: 12px;">青春不常在，抓紧谈恋爱</div>
 ```

### 内部样式表(嵌入式)

> 内部样式表(内联样式表)将样式声明在HTML文档的`<style>`标签中

```css
<style>
div {
color: red;
font-size: 12px;
}
</style>
```

> 注意:
>
> - `<style>`标签理论上可以放在HTML文档的任何地方，但一般放在`<head>`标签的尾部
> - 这种方式代码结构清晰，方便易用，但是并没有实现结构与样式完全分离

### 外部样式表(外链式)

> 外部样式表将样式声明在单独的CSS文件中，通过`<link>`标签将样式引入，这种方式实现了样式与结构的完全分离

```css
<link rel="stylesheet" href="css文件路径">
```

| 属性 |                             作用                             |
| :--: | :----------------------------------------------------------: |
| rel  | 定义当前文档与被链接文档的关系，“stylesheet”表示被链接的文件是一个样式表文件 |
| href |                   定义所链接外部文件的URL                    |

## 基础选择器

> 基础选择器是由单个选择器组成的，包括:标签选择器，类选择器，id选择器，通配符选择器

### 标签选择器(元素选择器)

> 标签选择器(元素选择器)通过HTML标签名(或者说HTML元素类型)进行选择，为页面中的某一种标签指定统一的CSS样式

```css
标签名 {
    属性1:属性值1;
 	属性2:属性值2;
}
```

### 类选择器

> 类选择器通过HTML元素的class属性进行选择，为具有同一类属性的元素添加样式

```css
/*选择某一种标签的某一类*/
标签.类名 {
    属性1: 属性值1;
	...
}
/*或*/
/*选择所有标签的某一类*/
.类名 {
	属性1: 属性值1;
	...
}
```

> 注意:
>
> - 元素的类名要保证见名知意，不要使用纯数字，中文命名，尽量使用英文字母加中横线命名
> - 一个元素可以具有多个类名，多个类名之间要用空格隔开，这些类名均可以独立的通过类选择器为元素附加样式

### Id选择器

> Id选择器通过HTML元素的Id属性进行选择，元素的Id必须是唯一的，可以为页面中唯一的元素附加样式

```css
#id {
	属性1: 属性值1;
	...
}
```

> 注意:
>
> - id 选择器一般用于页面唯一性的元素上，经常和 JavaScript 搭配使用。
> - 规范上所有元素的Id必须唯一，但是就算Id重复也不会报错，不过请杜绝这样做
> - 为元素附加id属性或类属性以使它匹配某一选择器的做法有时会被形象的表述为`调用选择器`

### 通配符选择器

> 通配符选择器用`*`表示，意为选取页面中的所有元素。

```css
* {
	属性1: 属性值1;
	...
}
```

> 注意:
>
> - 通配符选择器 `*` 是自动应用到所有元素的，不需要在 HTML 中特意给元素加 class 或 id。

### 属性选择器

> 属性选择器通过元素的特定属性来选择元素，支持通配符匹配属性值

|  属性选择器   |                            作用                             |
| :-----------: | :---------------------------------------------------------: |
|     [att]     |                    选择具有att属性的元素                    |
|    E[att]     |                   选择具有att属性的E元素                    |
|  [att="val"]  |                  选择att属性值为val的元素                   |
| [att~="val"]  |           选择att属性值中包含val的元素(按词匹配)            |
| [att^="val"]  |                选择att属性值以val开头的元素                 |
| [att$="val"]  |                选择att属性值以val结尾的元素                 |
| [att*="val"]  |              选择att属性值包含val为子串的元素               |
| [att\|="val"] | 选择att属性值等于val或以val-开头的元素(分隔必须是连字符`-`) |



- 总结

|  基础选择器  |             作用             |       特点       |   使用情况   |         用法         |
| :----------: | :--------------------------: | :--------------: | :----------: | :------------------: |
|  标签选择器  |        选出相同的标签        |  不能差异化选择  |     较多     |    p {color:red}     |
|   类选择器   |   选出赋予同样类属性的标签   | 可以根据需求选择 |    非常多    |   .nav {color:red}   |
|   id选择器   | 一次只能选择一个对应id的标签 |  id属性是唯一的  | 一般和js搭配 |   #nav {color:red}   |
| 通配符选择器 |        选择所有的标签        |       全选       | 特殊情况使用 |    * {color:red}     |
|  属性选择器  |    选择具有特定属性的标签    |   根据需求选择   |     很少     | [title] {color:red;} |

## 复合选择器

> 复合选择器建立在基础选择器之上，一般是对基础选择器进行组合形成,包括:后代选择器，子选择器，并集选择器，伪类选择器等

### 后代选择器

> 后代选择器可以选择父元素中的所有符合规则的后代元素

```css
选择器1 选择器2 ... {
    
}
```

> 注意：
>
> - 选择器之间用空格隔开
> - 可以看作是多次筛选，最终选择的是最后的选择器所选择的元素
> - 组合的选择器可以是任意基础选择器

### 子选择器

> 子选择器可以选择父元素中所有符合规则的子元素，即最近一级的后代元素

```css
选择器1 > 选择器2 ... {

}
```

> 注意:
>
> - 只能选择子元素

### 并集选择器

> 并集选择器可以选择多个选择器结果的并集，通常用于集体声明

```css
选择器1,选择器2...{

}
```

### 伪类选择器

> 伪类选择器常用于在其他选择器基础通过伪类名称选出特定状态的元素，不同的元素具有不同的伪类名称

```
选择器1:伪类名称 {

}
```

> 注意:
>
> - 伪类选择器在选择器1选择出的元素的基础上，通过伪类名称再进行筛选

| 伪类名称 |                             效果                             |
| :------: | :----------------------------------------------------------: |
|  :link   |                     选择未被访问过的链接                     |
| :visited |                     选择已被访问过的链接                     |
|  :hover  |                    选择鼠标指针悬停的元素                    |
| :active  |             选择活动链接(鼠标按下但未松开的链接)             |
|  :focus  |                    选取获得焦点的表单元素                    |
|  :root   | 选择文档的根元素，在HTML中通常就是`<html>`标签,权重高于html标签选择器 |

> 注意:
>
> - 前四种属性均可以用于a标签，为了确保生效，依照LVHA的顺序声明:link->visited->hover->active

### 结构伪类选择器

> 结构伪类选择器与普通的伪类选择器格式一样，但是他根据文档结构选择元素

|      选择器      |                            含义                            |
| :--------------: | :--------------------------------------------------------: |
|  E:first-child   | 匹配父元素的第一个子元素E，如果第一个元素不是E类型，则不选 |
|   E:last-child   |                                                            |
|  E:nth-child(n)  |                                                            |
| E:first-of-type  |              匹配父元素符合E的子元素的第一个               |
|  E:last-of-type  |                                                            |
| E:nth-of-type(n) |                                                            |

> 注意:
>
> - 常用于选择父元素中的第几个孩子
> - 指定孩子的n中不仅可以填具体数字，还可以填n的表达式表示一系列数字，n从0开始
> - n还可以是关键字，even偶数，odd奇数
> - E可以省略不指定写为 :first-child，即选择父元素中的第一个任意类型的子元素
> - 不带type的选择器是将所有孩子排一个序号，选择第n个孩子；带type的选择器将指定类型的孩子排序号，选择第n个孩子

### 伪元素选择器

> 通过CSS创建一个元素而非通过HTML，从而简化HTML结构

```css
E::before /*在元素内部的前面插入内容*/
E::after /*在元素内部的后面插入内容*/
```

> 注意:
>
> - 伪元素通过CSS在元素内部创建一个行内元素
> - 这个被创建的元素在HTML文档树中是找不到的，所以称为伪元素
> - 伪元素选择器样式声明中必须声明content内容属性
> - 权重为0，0，0，1

# CSS元素显示模式

> 元素显示模式就是元素以什么方式进行显示，一般分为:块元素，行内元素和行内块元素。

## 块元素

```html
<h1></h1>
<p></p>
<div></div>
<ul></ul>
<ol></ol>
<li></li>
```

> 特点:
>
> - 块元素独占一行
> - 宽高以及内外边距可以控制
> - 宽度默认为父级元素的100%
> - 块元素可以看作一个盒子，里面可以放别的元素
>
> 注意:
>
> - `<h1>`和`<p>`等文字类块级元素主要用于存放文字，里面不能放其他块元素

## 行内元素

> 行内元素又叫内联元素

```html
<a>
<strong>
<b>
<em>
<i>
<del>
<s>
<ins>
<u>
<span>
```

> 特点:
>
> - 一行内可以放置多个行内元素
> - 无法设置宽高
> - 默认宽度就是本身内容的宽度
> - 行内元素只能内嵌文本内容或其他行内元素
>
> 注意:
>
> - 链接里面不能再放链接
> - 特殊情况链接 <a> 里面可以放块级元素，但是给 <a> 转换一下块级模式最安全

## 行内块元素

> 行内元素中有几个特殊的元素，同时具有块元素和行内元素的特点

```
<img>
<input>
<td>
```

> 特点:
>
> - 和相邻行内元素（行内块）在一行上，但是他们之间会有空白缝隙。一行可以显示多个（行内元素特点）。
> - 默认宽度就是它本身内容的宽度（行内元素特点）。
> -  高度，行高、外边距以及内边距都可以控制（块级元素特点）。

## 显示模式转换

> 为了灵活的布局，可以通过样式声明的display属性自由的转换元素的显示模式,转换之后，元素就可以拥有对应模式的特性

```css
display : block 
display : inline 
display : inline-block
```

# CSS属性

> 通过修改属性值来修改元素的显示样式

## 字体属性

> 字体属性均为`font-att`格式，可以修改元素内文本的字体系列，大小，粗细，风格

- font-family

> 定义字体系列

```css
font-family: 'Microsoft YaHei' , tahoma,arial , 'Hiragino Sans GB'
```

> 注意:
>
> - 如果设置多个字体系列，不同系列之间要用`,`隔开，浏览器会根据字体顺序从前到后检查，如果电脑带有则显示
> - 如果有空格隔开的多个单词组成的字体,加单引号

- font-size

> 定义字体大小，通常使用px(像素)作为单位

```css
font-size : 16px
```

> 注意：
>
> - 谷歌浏览器默认的文字大小为16px

- font-weight

> 定义字体粗细，属性值可以为数字，也可为关键字

```css
font-weight : normal
```

| 属性值  |                  含义                  |
| :-----: | :------------------------------------: |
| normal  |                 默认值                 |
|  bold   |                  粗体                  |
| 100-900 | 400等同于normal，700等同于bold，无单位 |

- font-style

> 设置字体风格

```css
font-style : normal
```

| 属性值 |  含义  |
| :----: | :----: |
| normal | 默认值 |
| italic |  斜体  |

- font

> 字体复合属性，通过一个属性设置字体的全部样式

```css
font : font-style font-weight font-size/line-height font-family
```

> 注意:
>
> - 各个属性要以空格分隔，必须按照上述顺序
> - font-size和font-family属性是必须的，其他属性可以省略(取默认值),否则font属性将不起作用

## 文本属性

> 文本属性大多为`text-att`格式，用于定义一段文本的样式，如文本颜色，文本对齐方式，装饰文本，文本缩进，行间距

- color

> 定义文本颜色

```css
color : #FF0000
color : red
color : rgb(255,0,0)
color : rgb(100%,0%,0%)
```

> 注意:
>
> - 属性值可以通过16进制数，颜色关键词或rgb指定

- text-aligen

> 定义文本内容的水平对齐方式

```css
text-aligen : center
```

| 属性值 |      含义      |
| :----: | :------------: |
|  left  | 左对齐(默认值) |
| right  |     右对齐     |
| center |    居中对齐    |

- text-decoration

> 定义文本的修饰内容

```
text-decoration : none
```

|    属性值    |          含义           |
| :----------: | :---------------------: |
|     none     |     无装饰线(默认)      |
|  underline   | 下划线(`<a>`自带下划线) |
|   overline   |         上划线          |
| line-through |         删除线          |

- text-indent

> 定义文本的第一行缩进距离，一般单位为px(像素)或em

```css
text-indent : 2em
```

> 注意:
>
> - em是一个相对单位，即当前元素中font-size(一个文字)的大小，如果当前元素没有设置文字大小，则会按照父元素文字大小。
> - 缩进距离可以为负值

- line-height

> 定义文本行高,单位为px(像素)，如果不带单位则表示当前元素文字大小的n倍

```
line-height : 20px
```

> 注意:
>
> - 可以利用css的继承性，通过在父元素中指定不带单位的行高，使子元素根据自己的文字大小自动调整行高

## 背景属性

> 背景属性用于设置背景颜色、背景图片、背景平铺、背景图片位置、背景图像固定等，一般为`background-att`格式

- background-color

> 定义了元素的背景颜色

```css
background-color : transparent(透明色，为默认值)
background-color : rgba(red,green,blue,alpha)
background-color : rgb()
```

> 注意:
>
> - 背景色可以设置透明度，取值为0-1,一般会将如0.3的0省略掉，写为`.3`。

- background-image

> 定义元素的背景图像，常用于精灵图

```css
background-image : none|url(图像url)
```

> 注意:
>
> - 图像url不需要加引号

- background-repeat

> 设置背景图像的平铺方式

```
background-repeat : repeat
```

|  属性值   | 含义 |
| :-------: | :--: |
|  repeat   |      |
| no-repeat |      |
| repeat-x  |      |
| repeat-y  |      |

- background-position

> 定义图片在背景中的位置,属性值可以设置为精确单位或方位名词

```css
background-position : x y
```

|  参数值  |                         说明                          |
| :------: | :---------------------------------------------------: |
| 精确单位 |        百分数或浮点数和单位标识符组成的长度值         |
| 方位名词 | x可以取left\|center\|right,y可以取top\|center\|bottom |

> 注意:
>
> - 如果两个值都是方位名词，则两个值前后顺序无关
> - 如果只指定了一个方位名词，另一个省略，则默认为居中对齐
> - 如果只指定一个精确单位，另一个省略，则默认y垂直居中
> - 如果x与y使用不同的方式指定，则第一个值为x，第二个值为y
> - 图片与元素左上角对齐，以左上角为原点，向右为x轴正方向，向下为y轴正方向

- background-attachment

> 设置背景图像固定或随着页面滚动

```css
background-attachment : scroll|fixed
```

- background复合属性

> 将所有背景样式设定写在background符合属性中，

```css
background : background-color background-image background-repeat background-attachment background-position
```

> 注意:
>
> - 图像复合属性书写顺序没有硬性要求

## 界面样式属性

> 界面样式属性用于控制与用户交互相关的界面表现，从而提升用户体验。

- 鼠标样式

> 设置用户鼠标指针在元素上的样式

```css
cursor : pointer
```

|   属性值    |    描述    |
| :---------: | :--------: |
|   default   | 小白(默认) |
|   pointer   |    小手    |
|    move     |    移动    |
|    text     |    文本    |
| not-allowed |    禁止    |

- outline(轮廓线)

> 设置键盘聚焦时的边框样式，尤其是表单元素中的输入框

```css
/*设置为0或none后，聚焦后，没有轮廓线*/
outline : 0 | none 
```

- resize

> 控制是否允许用户调整元素大小，尤其是文本域

```css
resize : none
```

## 列表属性

> 用于设置列表元素的相关样式

- list-style

> 设置列表元素的风格

```css
list-style : none
```



## 其他属性

- vertical-align

> 加在行内或行内块元素上，设置行内元素或行内块元素以及文字基于行框的垂直对齐方式

```css
vertical-align : baseline | top | middle | bottom
```

|  属性值  |                             描述                             |
| :------: | :----------------------------------------------------------: |
| baseline | 默认值，基线对齐，行内元素的基线与其父容器（行框）的基线对齐。 |
|   top    |            元素顶部与其父容器（行框）的顶部对齐。            |
|  middle  |            元素基线与其父容器（行框）的中线对齐。            |
|  bottom  |            元素底部与其父容器（行框）的底部对齐。            |

> 注意:
>
> - **行框（line box）**：每一行文字或行内元素都会创建一个行框，这个行框包括了所有行内元素的高度（包括 `line-height`）。
> - **基线**：行内元素通常与行框的基线对齐，基线是每个字符的底部对齐参考线。

## 元素显示与隐藏

> CSS中有多种属性可以控制元素的显示与隐藏,如`display`,`visibility`,`overflow`

- display

> 通过display属性隐藏元素后，元素不再占有原来的位置

```css
display : none | block
```

> 注意:
>
> - display除了控制元素显示隐藏外，还可以用来改变元素显示模式
> - block除了表示块元素外，同时还有显示元素的意义

- visibility

> 通过visibility属性隐藏元素后，元素继续占有原本的位置

```css
visibility : visible | hidden
```

- overflow

> `overflow` 属性用于控制当元素内容溢出其容器时的显示方式。

```css
overflow : visible | hidden | scroll | auto
```

| 属性值  |                 描述                 |
| :-----: | :----------------------------------: |
| visible |     不做处理，在外部正常显示内容     |
| hidden  |           溢出部分内容隐藏           |
| scroll  | 溢出部分滚动条显示，且一直显示滚动条 |
|  auto   | 有溢出内容时显示滚动条，没有时不显示 |



# CSS三大特性

## 继承性

> 子元素会继承父元素的一些样式

- 特点

1. 恰当地使用继承可以简化代码，降低 CSS 样式的复杂性
2. 子元素可以继承父元素的样式（text-，font-，line-这些元素开头的可以继承，以及color属性）

## 层叠性

> 层叠性用于解决样式冲突问题，如果多个选择器同时为一个元素的同一个属性设置样式，后设置的样式可以覆盖先设置的样式

- 原则

1. 样式冲突，遵循的原则是就近原则，哪个样式离结构近，就执行哪个样式
2. 样式不冲突，不会层叠

## 优先级

> 当多个选择器选择到同一个元素时，当他们有相同的样式声明，就会有优先级的产生。

|            选择器             |  权重   |
| :---------------------------: | :-----: |
|          继承 和  *           | 0,0,0,0 |
|          标签选择器           | 0,0,0,1 |
|     类选择器和伪类选择器      | 0,0,1,0 |
|           ID选择器            | 0,1,0,0 |
|       行内样式 style=""       | 1,0,0,0 |
| !importrant(在样式声明中添加) | 无穷大  |

> 注意:
>
> - 如果权重相同，则执行层叠性
> - 如果权重不同，则根据优先级
> - 复合选择器的权重计算即为基本选择器简单叠加
> - !important的使用:  color: green !important;

- 层叠性和优先级注意点

> 无论是层叠行还是优先级，都只会作用在样式冲突的属性上，如果两个选择器之间无样式冲突的情况下，选择了同一个元素，则这两个选择器都会生效

> 盒子模型就像是一个“盒子外套”，它把 HTML 元素包起来，并给我们提供控制这个盒子外部（margin）、边框（border）、内部空隙（padding）和内容（content）的能力，从而实现网页排版和布局。

# 网页布局的三种方式

> 网页布局的本质就是用CSS摆放盒子到对应的地方，CSS提供了三种布局方式(即摆放盒子的方式):普通流(标准流)，浮动，定位。实际开发中，一个页面的搭建通常会将三种布局方式混合使用

## 标准流(普通流/文档流)

> 标准流就是按照标签规定好的默认方式排列

- 规则

1. 块级元素独占一行，从上向下顺序排列
2. 行内元素按照顺序，从左到右排列，到达边缘自动换行

## 浮动

> 浮动布局即为元素设置float属性，使其从标准流中脱离
>
> float 属性用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘。这是什么意思

```css
float : none | left | right
```

| 属性值 |      含义      |
| :----: | :------------: |
|  none  | 不浮动(默认值) |
| lefat  |    向左浮动    |
| right  |    向右浮动    |

### 浮动特性

1. 浮动的元素会脱离标准流，不再占有原先的位置，按照浮动的规则移动到指定位置(脱标)
2. 浮动的元素会在一行内显示并且顶部对齐，互相紧紧贴在一起，如果父级元素一行装不下这些元素，会自行另起一行
3. 浮动元素会具有行内块的特性，但是浮动元素都属于块级元素，所以一些行内或行内块属性对他无效

> 注意:
>
> - 浮动的元素只会影响浮动盒子后面的标准流元素，不会影响前面的标准流

### 清除浮动

> 浮动元素会脱离标准流，如果父元素并未设置高度而是依靠子元素自行撑开，此时浮动元素可能导致父元素高度塌陷，影响下面的布局。
>
> 此时需要清除浮动，清除浮动的本质就是清除浮动元素造成的影响
>
> 清除浮动之后，父级元素就会根据浮动的盒子自动检测高度。
>
> 清除浮动的主要策略为闭合浮动

- clear属性

> 清除浮动的方法都用到了这个属性

```css
clear : both
```

| 属性值 |                  含义                  |
| :----: | :------------------------------------: |
|  left  | 不允许左侧有浮动元素(清除左侧浮动影响) |
| right  | 不允许右侧有浮动元素(清除右侧浮动影响) |
|  both  |        同时清除左右两侧浮动影响        |

- 额外标签法

> 额外标签法也称为隔墙法，是 W3C 推荐的做法。
>
> 额外标签法会在浮动元素末尾添加一个空的标签。例如 `<div style="clear:both"></div>`，或者其他标签（如`<br />`等）。

```html
 <div class="father">
    <div class="box1"></div>
    <div class="box2"></div>
    <div style="clear:both"></div>
 </div>
```

> 优点：通俗易懂，书写方便
>
> 缺点： 添加许多无意义的标签，结构化较差
>
> 注意： 要求这个新的空标签必须是块级元素。

- 为父级元素添加overflow属性

```
overflow : hiddent | scroll | auto
```

> 优点:代码简洁
>
> 缺点:无法显示溢出的部分
>
> 注意:

- :after伪元素法

> 额外标签法的升级版，为父元素添加

```css

}
.clearfix { /* IE6、7 专有 */

}
```

>  优点：没有增加标签，结构更简单
>
>  缺点：照顾低版本浏览器

- 双伪元素法

```

}
```

>  优点：代码更简洁

## 定位

> 定位是通过CSS 的`position`属性和相关的定位属性来控制元素的位置。它决定了元素相对于其正常位置、父元素、或浏览器窗口的偏移。
>
> 定位需要设置定位模式与边偏移，定位模式指定元素在文档中的定位方式。边偏移则决定了该元素的最终位置

### 定位特性

1. 行内元素添加绝对定位或固定定位，可以设置宽高
2. 块元素添加绝对定位或固定定位，如果不设置宽高，默认大小为内容大小

> 原因:绝对定位与固定定位会使元素脱标

#### 定位模式

> 定位模式决定元素的定位方式 ，它通过 CSS 的` position` 属性来设置.

|  属性值  |   含义   |
| :------: | :------: |
|  static  | 静态定位 |
| relative | 相对定位 |
| absolute | 绝对定位 |
|  fixed   | 固定定位 |
|  sticky  | 粘性定位 |

- 静态定位

> 静态定位是元素默认的定位方式，即无定位

```css
position : static
```

> 注意:
>
> - 静态定位按照标准流特性摆放元素，无边偏移

- 相对定位

> 元素相对于其原始位置进行定位。它保留在文档流中的位置，移动是基于该位置的偏移。

```css
position : relative;
top : 20px;
left : 20px;
```

> 注意:
>
> - 当相对定位的元素与标准流元素重叠时，相对定位的元素会完全压住标准流元素

- 绝对定位

> 元素相对于最近的定位元素（非 `static` 的父元素）进行定位。如果没有定位元素，则相对于文档的 `<html>` 元素（即整个页面）。

```css
position: absolute;
  top: 50px; /* 相对于最近的定位父元素，向下 50px */
  right: 10px;
```

> 注意:
>
> - 绝对定位是脱离标准流的，不再占有原本的位置
> - 子绝父相:因为父级需要占有位置，因此是相对定位， 子盒子不需要占有位置，则是绝对定位.当然，子绝父相不是永远不变的，如果父元素不需要占有位置，子绝父绝也会遇到。

- 固定定位

> 元素相对于浏览器窗口（视口）进行定位，而不是相对于父元素。即使页面滚动，元素的位置也不会改变。

```css
position: fixed;
  top: 0;
  left: 0; /* 固定在页面左上角 */
```

> 注意:
>
> - 固定定位也是脱离标准流的，不再占有原本的位置

- 粘性定位	

> 元素相对于浏览器窗口（视口）进行定位，但是元素仍占有原本的位置

```css
position : sticky;
top : 10px
```

> 注意:
>
> - 使用粘性定位时，必须添加边偏移才能生效

#### 边偏移

> 边偏移即是通过`top`,`bottom`,`left`,`right`属性设置元素相对于其正常位置、父元素、或浏览器窗口的偏移，可以是像素值，也可以是相对于父元素的百分比

| 边偏移属性 |                          描述                          |
| :--------: | :----------------------------------------------------: |
|    top     | 顶端侧偏移量，定义元素相对于父元素(或自己)上边线的距离 |
|   bottom   |  底部偏移量，定义元素相对于父元素(或自己)下边线的距离  |
|    left    |  左侧偏移量，定义元素相对于父元素(或自己)左边线的距离  |
|   right    |  右侧偏移量，定义元素相对于父元素(或自己)右边线的距离  |

> 注意:
>
> - 如果对于同一个定位元素同时设置了left和right，此时left生效，同理，同时设置top和bottom，top生效

#### 定位堆叠顺序

> z-index属性控制层叠定位元素控制的是元素在 **z 轴方向（垂直屏幕）** 上的显示顺序

```css
z-index : 数字
```

> 注意:
>
> - 数值可以是正整数，负整数或0，默认为auto，数值越大，显示顺序越靠前
> - 值为auto(默认)情况下，按照书写顺序，后来居上
> - 只有定位的盒子受z-index影响

## 三大布局方式注意点

1. 浮动元素与绝对定位和固定定位元素不会触发外边距合并合并的问题
2. 浮动元素只会压住标准流的盒子，不会遮盖标准流盒子的内容，这些内容会环绕浮动元素显示(浮动最初产生的目的是做文字环绕效果的);但是绝对定位和相对定位元素会完全压住盒子

# CSS转换

> CSS3中具有颠覆性的特征之一，通过转换，可以实现元素的位移，旋转和缩放等效果

## 改变转换中心点

> 通过transform-origin改变元素转换时的中心点

```
transfrom-origin:x y;
```

> 注意:
>
> - 默认中心点为(50%,50%)
> - x与y可以是像素与百分比，也可以是方位名词(top bottom left right center)

## 移动(translate)

> 改变元素在页面中的配置类似定位

```
transform: translate(x,y);
transform: translateX(x);
transform: translateY(y);
```

> 注意:
>
> - 位移时以盒子左上角为原点，向右为X轴正方向，向下为Y轴正方向
> - translate不会影响其他盒子的位置
> - 参数可以为像素或其他相对单位，也可以是百分比，此时百分比是相对于自身宽度和高度的百分比
> - translate对行内标签无效果

## 旋转(rotate)

> 在二维平面内顺时针或逆时针旋转

```
transform: rotate(度数)
```

> 注意:
>
> - 度数单位为deg，必须跟单位
> - 角度为正时，顺时针；为负时，逆时针
> - 默认旋转轴为元素的中心点

## 缩放(scale)

> 以中心点为原点，将元素放大或缩小

```
transform:scale(x,y)
```

> 注意:
>
> - x与y是缩放的倍数
> - 可以只写一个参数，只写一个参数时，代表两个参数一样
> - 缩放不影响其他盒子

## 转换综合写法

```
transform: translate() rotate() scale()
```

> 注意：
>
> - 书写顺序会影响转换效果，因为旋转会改变坐标轴方向
> - 一般将位移放在最前面

# flex布局

- justify-content

  > 定义了项目在主轴上的排列方式

  | flex-start    | 默认值 从头部开始 如果主轴是x轴，则从左到右 |
  | ------------- | ------------------------------------------- |
  | flex-end      | 从尾部开始排列                              |
  | center        | 在主轴居中对齐（如果主轴是x轴则 水平居中）  |
  | space-around  | 平分剩余空间                                |
  | space-between | 先两边贴边 再平分剩余空间（重要）           |

- flex-wrap

  > 设置子元素是否换行，默认情况下，项目都排在一条线（又称”轴线”）上。当子元素摆放不开时，会自动缩小子元素大小

  | nowrap | 默认值，不换行 |
  | ------ | -------------- |
  | wrap   | 换行           |
  |        |                |

- align-items

  > 设置侧轴上的子元素排列方式在子项为单行的时候使用

  |            |                                        |
  | ---------- | -------------------------------------- |
  | flex-start | 从上到下                               |
  | flex-end   | 从下到上                               |
  | center     | 挤在一起居中（垂直居中）               |
  | stretch    | 拉伸 （默认值，子盒子给高度后不生效 ） |

- align-content

  > 设置侧轴上的子元素的排列方式（多行），在单行下是没有效果的。

  |               |                                        |
  | ------------- | -------------------------------------- |
  | flex-start    | 默认值在侧轴的头部开始排列             |
  | flex-end      | 在侧轴的尾部开始排列                   |
  | center        | 在侧轴中间显示                         |
  | space-around  | 子项在侧轴平分剩余空间                 |
  | space-between | 子项在侧轴先分布在两头，再平分剩余空间 |
  | stretch       | 设置子项元素高度平分父元素高度         |

  > 注意:
  >
  > - 此时父元素必须添加flex-wrap，并设置为wrap。

## CSS生成三角形

## html元素与body元素的区别

- 背景色设置

  > 根据 CSS 规范：**浏览器会默认将 `<html>` 的背景色渲染到整个视口**，不受 `html { height }`、`overflow` 限制的影响。所以当子元素内容无法填充整个视口时，html元素的背景色会铺满剩余部分。
  >
  > 而body元素的背景色只会根据自己的高度与宽度渲染。

## viewport

> 视口，表示浏览器整个可视窗口，vw为视口宽度，是一个长度单位，即viewport width，1vw表示视口宽度的1%;vh，viewport height,视口高度，1vh标识视口高度的1%.。使用这两个单位时，元素实际像素值会根据浏览器窗口的大小变化动态变化
>
> `vw` 是动态的，会随着视口的变化而变化。因此，如果浏览器窗口被缩小，使用 `vw` 单位的元素也会缩小，反之亦然。
>
> `vw` 是一个相对单位，适合用于布局、字体和一些需要根据视口变化的元素。
>
> `vw` 单位可以使得元素和文字的尺寸随着浏览器窗口的宽度变化而自动调整，非常适合响应式设计。

## CSS自定义属性

> 又称CSS变量，允许在 CSS 中定义一个可重用的值，然后在其他地方使用它。

- 使用

  > 自定义属性通常在CSS选择器中进行定义，通常是在:root选择器中定义全局变量，这时可以在整个文档中使用。定义之后，可以在作用范围内通过var(变量名)的方式使用

  ```css
  
  :root {
    --bg: #35ad4f;
  }
  
  
  body {
    background-color: var(--bg);  /* 使用变量 */
  }
  
  ```

  > 注意:
  >
  > - 所有CSS变量都需要以`--`开头标识

- 继承特性

  > 在父元素中定义的CSS变量，会被所有子元素自动继承，所以在:root选择器中定义的变量会被所有元素继承

## rem

> 相对单位，表示"root em",即相对于根元素`<html>`的字体大小(font-size)的单位

```
html {
  font-size: 16px;
}

p {
  font-size: 1rem;   /* 等于 16px */
  margin-bottom: 2rem;  /* 等于 32px */
}

```

> 注意:
>
> - 浏览器默认根元素字体大小为16px，则默认1rem = 16px

## 阴影相关属性

> css提供了设置文字阴影，边框阴影，盒子阴影的属性


> 注意:

|     属性      |                      含义                       | 是否必须 |
| :-----------: | :---------------------------------------------: | :------: |
|   offset-x    |      阴影在水平方向上的偏移(向右为正偏移)       |   必填   |
|   offset-y    |      阴影在垂直方向上的偏移(向下为正偏移)       |   必填   |
|  blur-radius  |  模糊距离（越大越模糊,值为像素或其他相对单位）  |   可选   |
| spread-radius | 扩展距离（影响阴影大小,值为像素或其他相对单位） |   可选   |
|     color     |                    阴影颜色                     |   可选   |
|     inset     |             是否为内阴影(值为inset)             |   可选   |

> 注意:
>
> - spread-radius为0时，阴影的大小等于元素本身的大小，为正时，阴影变大，为负时，阴影变小
